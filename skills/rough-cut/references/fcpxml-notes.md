# FCPXML: что генерим и известные грабли

`export_fcpxml.py` выдаёт FCPXML 1.9 для импорта в DaVinci Resolve (`File → Import → Timeline → Import AAF, EDL, XML…`).

## Что внутри

- `<!DOCTYPE fcpxml>` + `<fcpxml version="1.9">` — Resolve требует DOCTYPE.
- Все времена — рациональные, выровненные по границе кадра: `frames*den/num` секунд от `frameDuration=den/num` (например `3003/30000s` для NTSC). Невыровненное время — самая частая причина «Resolve молча съел, но клипы съехали».
- Резы «мягкие»: каждый keep-сегмент — `<asset-clip>` со `start` (точка в исходнике) и `duration`; исходный файл не тронут, границы двигаются в Resolve.
- Отдельный звук (OBS) — connected clip `lane="-1"` внутри каждого видеоклипа, `start` сдвинут на `sync.offset_s`.
- Мультикам — второе видео connected clip `lane="1"` с теми же резами.
- **speedup — обычный клип + маркер** `SPEEDUP ×N (rough-cut): <причина>`. Ретайм пользователь делает в Resolve сам (клик по клипу → Change Clip Speed). Кодировать `<timeMap>` не стали: импорт ретайма из FCPXML в Resolve ненадёжен, а молча сломанная скорость хуже маркера.

## Грабли

- Пути к исходникам — абсолютные `file://` URI. Перенёс файлы после экспорта → Resolve попросит relink, это нормально.
- Resolve игнорирует `event/project` имена при импорте таймлайна — таймлайн появится как «rough-cut».
- Если импорт падает с ошибкой схемы — записать сюда точный текст ошибки и версию Resolve, затем чинить экспортёр (частое: отсутствие DOCTYPE, `start` за пределами `duration` asset-а).
