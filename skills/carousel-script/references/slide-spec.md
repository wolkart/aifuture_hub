# slide-spec.md — схема per-slide спека

Центральный контракт карусели. На эту схему ссылаются `SKILL.md` и все платформенные брифы.

---

## Схема полей слайда

Каждый слайд описывается семью полями:

| Поле | Тип | Описание |
|---|---|---|
| `№` | число | Порядковый номер слайда в карусели (1, 2, 3 …) |
| `роль` | enum | Функция слайда: `обложка` / `ценность` / `CTA` |
| `заголовок` | строка | Главная фраза слайда — крупный акцент |
| `текст` | строка | Тело слайда: пояснение, аргумент, список пунктов |
| `подача` | enum | Количество и размер текста на карточке — см. словарь ниже |
| `лейаут` | enum | Структурный шаблон карточки — см. словарь ниже |
| `визуал-направление` | строка | Описание фоновой иллюстрации / фото; одновременно промпт для image-gen |

---

## Словарь «подача» (text-treatment)

Управляет тем, сколько текста помещается на карточке и каким кеглем он набирается.

| Значение | Описание |
|---|---|
| `мега-текст` | Текст на всю карточку огромными буквами (≈ 3 слова), бренд-значок в углу. Максимальный визуальный удар, минимум слов. |
| `заголовок+треть` | Заголовок крупно + текст занимает ~1/3 высоты карточки. Баланс между читаемостью и воздухом. |
| `плотный-текст` | Компактный набор, слов больше. Подходит для чеклистов, сравнений, нумерованных шагов. Кегль уменьшается, строки ужимаются. |

---

## Словарь «лейаут»

Стабильный набор шаблонов. Пользователь маппит их 1:1 на свои готовые Figma-компоненты.

| Значение | Характер |
|---|---|
| `обложка` | Полноэкранный заголовок + подзаголовок + имя автора / бренд |
| `большая-цифра` | Акцентная цифра или процент занимает половину карточки, рядом — пояснение |
| `цитата` | Крупная цитата в кавычках, мелко — источник или автор |
| `чеклист` | Нумерованный или маркированный список; строки одинакового веса |
| `текст+визуал` | Левая / верхняя зона — текст, правая / нижняя — иллюстрация или фото |
| `сравнение` | Две колонки: «до / после», «вариант A / вариант B» и т.п. |

---

## Правило fits-card

> **Текст ДОЛЖЕН помещаться на карточке при выбранной подаче.**
> Формат по умолчанию — **4:5, 1080 × 1350 px**.
> Это правило важнее любого счётчика слов.

Практические следствия:
- При подаче `мега-текст` — не больше 3–5 слов суммарно (заголовок + текст вместе).
- При подаче `заголовок+треть` — заголовок до 7 слов, текст до 25–30 слов.
- При подаче `плотный-текст` — текст до 50–60 слов; если больше — дробить на два слайда.
- Если контент не вписывается в одну карточку, скилл разбивает его на дополнительный слайд с ролью `ценность`.

## Пример заполненного спека (3 слайда)

### Слайд 1

```
№: 1
роль: обложка
заголовок: "5 ошибок, которые убивают вашу воронку"
текст: "Проверьте, нет ли у вас хотя бы одной"
подача: заголовок+треть
лейаут: обложка
визуал-направление: тёмно-синий градиентный фон, тонкая диагональная сетка, неоновый акцент — оранжевая линия снизу
```

### Слайд 2

```
№: 2
роль: ценность
заголовок: "Ошибка № 1: нет оффера на первом экране"
текст: "Первый экран — это обложка карусели в ленте. Если за 2 секунды непонятно, что внутри, — люди листают дальше. Оффер должен быть явным."
подача: заголовок+треть
лейаут: текст+визуал
визуал-направление: схематичный смартфон с пустым экраном, стрелка «пользователь уходит», плоская иллюстрация в оттенках синего
```

### Слайд 3

```
№: 3
роль: CTA
заголовок: "Сохрани и проверь воронку сегодня"
текст: "Подпишись, чтобы получить чеклист из 12 пунктов"
подача: мега-текст
лейаут: обложка
визуал-направление: яркий оранжевый фон, крупная стрелка вниз, бренд-значок в правом верхнем углу
```

---

## Заметки

- `визуал-направление` — это одновременно промпт для image-gen (Midjourney, Ideogram, DALL-E и др.). Формулируй описание так, чтобы его можно было вставить в генератор напрямую.
- `лейаут` — стабильный словарь: не расширяй его внутри карусели. Если нужен новый шаблон — добавь в словарь отдельно и обнови Figma-компонент.
- Поля `заголовок` и `текст` разделены намеренно: это разные типографические зоны на карточке с разным кеглем и весом.
