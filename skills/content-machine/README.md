# content-machine — где применять

Тонкий оркестратор: единая дверь «идея → готовый контент». По входу определяет источник идеи, делегирует готовым скиллам хаба и доводит до рилс-сценария и/или карусели. Сам контент не создаёт — только маршрутизирует (DRY).

## Когда срабатывает

- «Сделай контент про…», «запусти контент-машину», «придумай и собери рилс/карусель», «из этого сделай контент» — когда явно не указан один конкретный скилл.

НЕ для прямого вызова одного скилла (тогда зови его напрямую), не для постинга/доставки (это слой Hermes).

## Источники (роутер)

| Вход | Делегат |
|---|---|
| ссылка `instagram.com/p\|reel\|reels/…` | `instagram-downloader` |
| `@handle` / «разбери нишу» | `instagram-analytics` |
| «о чём снять» / «найди тему» / «контент-план» | `ai-research` |
| своя тема/тезис | сразу к производству |

Формат (рилс / карусель / пост / оба) машина рекомендует и фиксирует в «Допущениях»; можно переопределить. Пост (текст под Telegram/Threads/X/LinkedIn) делегируется `post-hq`, который сам разруливает площадку × цель.

## Чекпойнты и headless

- Ресёрч и разведка ниши: показать кандидатов → ты выбираешь → производство.
- Своя идея и одиночный референс: сразу к производству.
- **Headless:** если в запросе уже задано источник + идея + формат — машина идёт без вопросов (путь для автоматического драйвера / Hermes).

## Зависимости

Делегирует готовым скиллам хаба: `ai-research`, `instagram-downloader`, `instagram-analytics`, `reels-script`, `carousel-script`, `post-hq` (подключаются глобально через `skills/install-skills.ps1`). Своего `.env` у content-machine нет — конфиг живёт в суб-скиллах.

## Примеры вызова

```
сделай контент про то, как AI экономит время маркетологу
```

```
запусти контент-машину: найди тему про AI-агентов и собери карусель
```

```
вот рилс https://instagram.com/reel/XXXX — сделай из него контент под мою нишу
```

В ответ — строка-трасса `источник → идея → формат`, блок «Допущения» и финальный артефакт от делегата.
