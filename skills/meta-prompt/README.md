# meta-prompt — где применять

Скилл превращает сырую идею в готовый структурированный промпт. Полезен шире, чем «один промпт под одну задачу».

## Сценарии

1. **Отдельный промпт под задачу** — основной кейс. Кидаешь идею → получаешь готовый промпт.

2. **Ядро для нового скилла (рекурсия).** При создании скилла через `skill-creator` используй meta-prompt, чтобы сгенерировать содержательное «ядро» инструкции (роль, структура, ограничения, формат, примеры). Особенно мощно для контент-скиллов (сценарии Reels/Shorts, карусели для Instagram/LinkedIn), где сердце скилла — по сути отличный промпт.

   ⚠️ Вывод meta-prompt — это **сырьё, а не финальный `SKILL.md`**. Адаптируй:
   - `{{ПЛЕЙСХОЛДЕРЫ}}` и `<scratchpad>` → язык скилла (императивные инструкции для агента);
   - блок «когда спрашивать» → обычный диалог скилла;
   - вариативность (платформы, форматы) → выноси в `references/`;
   - триггер скилла живёт в `description`, а не в теле промпта.

   Маршрут: `skill-creator` (замысел, триггеры, тесты) → `meta-prompt` (ядро) → `skill-creator` (упаковка в `SKILL.md`, тесты, итерации).
