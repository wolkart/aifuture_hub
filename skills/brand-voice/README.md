# brand-voice — слой-фильтр голоса автора

Скилл упаковывает голос автора и универсальные принципы построения текста в единый переиспользуемый артефакт. Работает как слой-фильтр поверх любого написания постов: проводит входной материал через голос автора с минимальным вмешательством, сохраняя авторский замысел.

## Что делает

- **Лестница вмешательства** — степень правки зависит от входа: сырьё (строишь сам), полуфабрикат (достраиваешь), почти готовый текст (лёгкое касание). Чем ближе к финалу — тем меньше трогаешь.
- **Критика по светофору** — 🟢 сохранить · 🟡 склоняться к автору · 🔴 честно «так работать не будет» (молчать в красной зоне нельзя).
- **Telegram-вёрстка** — абзацы ~2 строки + пустая строка; это ритм, не догма: списки, шаги и цитаты не дробить.
- **Анти-шаблонизация** — словарь-регистр это иллюстрация, не чек-лист: характерность берётся из мысли и ритма, а не из вставленных маркеров.

## Когда срабатывает

«Напиши пост», «напиши пост в телегу», «оформи в моём голосе», «оформи в моём стиле», «доведи мой черновик», «причеши мой черновик», «перепиши под мой стиль» — или когда пользователь присылает сырьё / тезисы / каркас и просит сделать из этого пост.

## Что внутри

| Файл | Содержимое |
|------|-----------|
| `SKILL.md` | Рабочий контракт: шаг 0 (читать references), лестница входов, светофор, вёрстка, границы |
| `references/brand-voice.md` | Голос автора: кто говорит, тон, структура предложений, словарь-регистр, приёмы, юмор, эмодзи, вёрстка, запрещённое, библиотека образцов |
| `references/writing-principles.md` | Универсальные принципы сильного текста (§1–5) + §6 Граница редактуры (предохранитель) |

`references/brand-voice.md` — **живой документ, v0, докручивается** по мере кристаллизации голоса автора. На него могут ссылаться другие скиллы (например, `post-writer`) и штабы как на единый источник голоса.

## Что НЕ делает

Скилл не постит и не доставляет контент — это задача слоя рантайма (отдельный скилл / штаб). Не переписывает авторский замысел: только фильтрует подачу и голос.
