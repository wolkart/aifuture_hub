# hook-base — где применять

Скилл — **библиотека+резолвер хуков**: хранит сильные хук-паттерны по единой схеме и быстро отдаёт лучшие под конкретный ролик. Не пишет сценарии/посты и не анализирует профили — только хранит и подбирает хуки.

## Две операции

- **ПОЛОЖИТЬ** — диктуешь хук словами: «внеси хук „…"», «залетел ролик, хук был такой-то», «увидел классный хук, добавь». Скилл молча классифицирует по схеме (тип/ниша/формат/шаблон/когда работает/кросс-нишевость/источник/метрики), кладёт запись и отвечает одной строкой `Записал: …`. Скрапер (instagram-analytics) — лишь один из источников, не единственный.
- **ВЗЯТЬ** — просишь хук под тему: «дай хук под рилс про…», «подбери хук для карусели про…». Скилл фильтрует+ранжирует склад, подставляет шаблон под твою тему и отдаёт 2–3 кандидата с пометкой дефолта. Пусто по теме — честно говорит и отдаёт управление.

reels-script и carousel-script на шаге 2 резолва хука/обложки вызывают этот скилл (операция ВЗЯТЬ); встроенный хребет виральности остаётся фоллбэком.

## Где склад

- **Канон** — Obsidian-волт: `2-Контент/База хуков/` (путь из конвенции глобального CLAUDE.md). Листать удобно через вид `База хуков.base`. Отдельно от `Разведка/` — лучшее из разведки «повышается» в базу как чистая запись.
- **Фоллбэк** — если волта нет (харнес забрал другой человек): `references/patterns/` в репо с обезличенным примером-затравкой. Свой склад такой человек наполняет сам.

Схема записи, правила молчаливой классификации и конвенция имён — в [references/schema.md](references/schema.md).

## Пример вызова

```
внеси хук: "90% людей используют ChatGPT как поисковик"
```
```
дай хук под рилс про то, что вайб-кодинг без понимания опасен
```
