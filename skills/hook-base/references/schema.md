# Схема записи базы хуков

Один хук = один `.md`-файл с YAML-фронтматтером по этой схеме. Тело файла — живой пример хука.
Схема едина: и ручной приём, и будущий импорт из скрапера пишут записи в ТОЙ ЖЕ форме.

```yaml
---
тип: hook
тип_хука: отрицание        # провокация | неожиданность | интрига | отрицание | обещание
ниша: ИИ
формат: рилс               # рилс | карусель | универсальный
шаблон: "{X} не заберёт {Y} — заберёт тот, кто умеет с этим работать"
когда_работает: "страх потери работы из-за ИИ"
кросс_нишевость: да        # да | нет — переносится ли хук в другую нишу
источник: "ручной"         # URL ролика | "залетел, 120k просмотров" | "скаут <автор>" | "ручной"
метрики: { просмотры: null, охват: null }
---
ИИ не заберёт твою работу — её заберёт тот, кто умеет с ним работать
```

## Поля

- **тип** — пока всегда `hook`. (`script` зарезервирован на будущее, не используется.)
- **тип_хука** — ровно одно из: `провокация`, `неожиданность`, `интрига`, `отрицание`, `обещание`.
- **ниша** — кириллицей (`ИИ`, `фитнес`, `финансы`, …). По умолчанию — ниша автора (`ИИ`), если из текста не следует иное.
- **формат** — где хук работает: `рилс` (первая фраза ролика), `карусель` (заголовок обложки), `универсальный` (и там, и там).
- **шаблон** — обобщённый скелет хука со слотами `{X}`/`{Y}`; по нему резолвер подставляет конкретную тему. См. «Как строить шаблон».
- **когда_работает** — короткая фраза о боли/состоянии зрителя, при котором хук срабатывает.
- **кросс_нишевость** — `да`, если каркас переносится в другую нишу подстановкой; `нет`, если завязан на специфику ниши.
- **источник** — откуда хук: URL, описание «залетевшего» ролика с метрикой, `скаут <автор>`, или `ручной`.
- **метрики** — `просмотры`/`охват`, если известны; иначе `null`.

## Правила молчаливой классификации

Тип хука выводится из формы фразы (по маркерам), без расспросов:

| Маркеры в тексте | тип_хука |
|---|---|
| «не …, а …», «X не сделает Y», переворот ожидания через отрицание | отрицание |
| вызов, спор, «90% делают неправильно», обвинение/ставка | провокация |
| «секрет», «мало кто знает», «вот что скрывают», открытая петля | интрига |
| «X за Y минут», «без усилий», «бесплатно/open-source», результат-обещание | обещание |
| «оказывается», «на самом деле», контринтуитивный факт | неожиданность |

Если форма неоднозначна — выбери доминирующий маркер. Если ниша/смысл реально неясны — это единственный повод задать короткий вопрос.

## Как строить шаблон

Замени в живой фразе конкретику на слоты `{X}`, `{Y}`, сохранив каркас и ритм:

- `90% людей используют ChatGPT как поисковик` → `{доля} людей используют {инструмент} как {примитив}`
- `ИИ не заберёт твою работу — её заберёт тот, кто умеет с ним работать` → `{X} не заберёт {Y} — заберёт тот, кто умеет с этим работать`

Слотов 1–3, не больше: шаблон должен оставаться читаемым каркасом, а не дырявой заготовкой.

## Имя файла и номер

`hook-<типхука-translit>-<ниша-translit>-NN.md`, напр. `hook-otritsanie-ai-02.md`.

- Кириллица — только внутри фронтматтера; имя файла — латиницей (translit): `провокация→provokatsiya`, `неожиданность→neozhidannost`, `интрига→intriga`, `отрицание→otritsanie`, `обещание→obeshchanie`; `ИИ→ai`, `фитнес→fitnes`, `финансы→finansy`.
- `NN` — следующий свободный двузначный номер для пары тип_хука+ниша (`01`, `02`, …).

## Где склад

- **Канон** — Obsidian-волт: `2-Контент/База хуков/`. Путь волта берётся из конвенции (прописан в глобальном `~/.claude/CLAUDE.md`). Отдельно от `2-Контент/Разведка/` (там сырой вывод скрапера; лучшее оттуда «повышается» в базу как чистая запись).
- **Фоллбэк** — если волт не сконфигурирован (напр., харнес забрал другой человек): репо `skills/hook-base/references/patterns/` с обезличенным примером-затравкой. Свой склад такой человек строит сам.
