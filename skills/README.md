# Skills

Публичные скиллы, которыми я делюсь. Это **скиллы для Claude Code** (Anthropic): папка с файлом `SKILL.md`, которую Claude подхватывает и начинает использовать, когда задача под неё подходит.

Каждый скилл лежит в своей подпапке. Внутри — `SKILL.md` (инструкция, которую читает Claude) и краткий `README.md`: что делает и когда срабатывает.

## Как это устроено (важно для новичков)

Claude Code ищет скиллы в служебной папке: `~/.claude/skills/` (глобально, во всех проектах) или `<проект>/.claude/skills/` (только в одном проекте). **Просто скачать папку из этого репозитория недостаточно** — Claude не увидит её, пока скилл не окажется в одной из этих папок.

Чтобы файлы скилла жили в одном месте (в репозитории, где их удобно обновлять через `git pull`), но при этом Claude их находил, мы **не копируем**, а делаем **ссылку** (junction на Windows / симлинк на macOS·Linux) из `~/.claude/skills/<скилл>` на папку скилла в репозитории. Поправил репо — скилл сразу актуален, дублей нет.

## Как забрать и подключить скилл

1. **Забрать:** скачай репозиторий архивом (`Code → Download ZIP`) или склонируй:
   ```bash
   git clone https://github.com/wolkart/aifuture_hub.git
   ```
2. **Подключить (активировать):** создай ссылку на скилл в папке скиллов Claude. Проще всего — одним скриптом сразу все:
   ```powershell
   # Windows (без прав админа)
   powershell -ExecutionPolicy Bypass -File .\aifuture_hub\skills\install-skills.ps1
   ```
   ```bash
   # macOS / Linux / Git Bash
   bash ./aifuture_hub/skills/install-skills.sh
   ```
   Скрипт идемпотентен: подключает новые скиллы, уже подключённые не трогает — запускай повторно после каждого `git pull`.

Подробности (подключить один скилл вручную, привязать к одному проекту, что делать при переносе репозитория) — в [корневом README](../README.md#как-подключить-скиллы-к-claude-code).
