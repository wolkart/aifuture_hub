# Схема записи паттерна

Один хук или сценарий = один `.md`-файл с фронтматтером по этой схеме.
Встроенные сид-хуки — затравка; будущая внешняя база пишет записи в ТОЙ ЖЕ схеме.

```yaml
---
тип: hook            # hook | script
тип_хука: провокация # провокация | неожиданность | интрига | отрицание | обещание
ниша: ИИ             # ИИ | фитнес | финансы | ...
источник: "ручной"   # URL ролика или описание
метрики:             # если известны
  просмотры: null
  охват: null
когда_работает: "когда зритель считает, что делает всё правильно"
кросс_нишевость: да  # да | нет — можно ли перенести хук в другую нишу
---
```

Тело файла под фронтматтером — текст хука (или сценария).
