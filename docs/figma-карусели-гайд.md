# Карусели в Figma руками ИИ — простой гайд

Этот гайд объясняет **на пальцах**, как сделать так, чтобы Claude (ИИ) рисовал карусели для Instagram **прямо внутри Figma**: придумывал текст слайдов и сам собирал из них дизайн.

Работает **даже на бесплатном плане Figma**.

---

## Что вообще происходит (картинка в голове)

Чтобы магия заработала, три маленьких помощника «договариваются» между собой:

1. **Скилл `carousel-script`** — придумывает текст и раскладку слайдов.
2. **Мостик** — крошечная программа на твоём компьютере. Передаёт сообщения между Claude и Figma.
3. **Плагин внутри Figma** — получает сообщения от мостика и рисует слайды на холсте.

`Claude → мостик → плагин → дизайн в Figma`. Всё.

---

## Что понадобится (один раз поставить)

- **Figma Desktop** — приложение Figma (не браузер). Бесплатного плана достаточно.
- **Node.js** — бесплатная программа-«движок». Скачать: https://nodejs.org (кнопка LTS).
- **Bun** — ещё один «движок», нужен именно для мостика. Ставится одной командой (ниже).

> Node и Bun — это просто «проигрыватели», которые умеют запускать наши программки. Они ничего не меняют в системе, просто лежат и работают по запросу.

---

## Установка (делается один раз)

Открой **PowerShell** (синий терминал: жми «Пуск» → набери `PowerShell` → Enter).

**1. Поставь Bun** (если ещё нет):
```
npm install -g bun
```

**2. Скачай мостик** (в любую удобную папку, например в профиль пользователя):
```
git clone https://github.com/arinspunk/claude-talk-to-figma-mcp.git
```
Запомни путь к этой папке — он понадобится дальше (например `C:\Users\ИМЯ\claude-talk-to-figma-mcp`).

**3. Подключи мостик к Claude Code** — добавь MCP-сервер (одна команда):
```
claude mcp add claude-figma -- npx -y -p claude-talk-to-figma-mcp@latest claude-talk-to-figma-mcp-server
```
Проверка: `claude mcp list` — увидишь `claude-figma … ✓ Connected`.

---

## Подключение плагина в Figma (для ролика — самое важное)

**4. Импортируй плагин в Figma Desktop:**
- Открой Figma Desktop.
- Сверху: **Menu → Plugins → Development → Import plugin from manifest…**
- Выбери файл внутри скачанной папки: `src/claude_mcp_plugin/manifest.json`
- Готово — плагин теперь в списке (это разовое действие).

---

## Как пользоваться (каждый раз)

**5. Запусти мостик** и оставь окно открытым:
```
cd <путь-к-папке-мостика>
bun run src/socket.ts
```
Увидишь строку про `port 3055` — значит мостик работает.

> 💡 При первом запуске Windows может спросить разрешение фаервола для программы — **разрешай**. Мостик общается с плагином только **внутри твоего компьютера**, в интернет ничего не уходит.

**6. Открой плагин в Figma** (Plugins → Development → плагин из шага 4). Он покажет короткий **код-канал** — скопируй его. Открой любой пустой файл в Figma, куда будем рисовать.

**7. Перезапусти Claude Code** (закрой и открой заново) — чтобы он подхватил новый инструмент.

**8. Скажи Claude:**
> Подключись к Figma, канал `<твой-код>`, и собери карусель про <тему>.

Claude придумает слайды (через `carousel-script`) и начнёт рисовать их прямо в Figma. Останется проверить и поправить.

---

## Частые вопросы

- **«Выскочило окно Windows про разрешение» —** это фаервол спрашивает про мостик. Разрешай, это безопасно (работа только внутри твоего ПК).
- **«У меня бесплатная Figma» —** для этого способа подходит. Платный план нужен только для *официального* способа Figma; мы идём через бесплатный плагин-мостик.
- **«Что значит перезапустить Claude» —** просто закрыть приложение и открыть снова.
- **«Мостик и плагин обязательно держать включёнными?» —** да, пока рисуешь. Закрыл — связь пропала; открой снова по шагам 5–6.
