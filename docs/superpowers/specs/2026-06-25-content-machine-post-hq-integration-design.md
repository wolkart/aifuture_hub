# Дизайн: интеграция content-machine → post-hq + ai-research

**Дата:** 2026-06-25
**Статус:** утверждён, готов к плану
**Цикл:** роадмап post-hq — авто-связка «нашёл новость/идею → написал пост» (наряду с reels/carousel).

## Цель

Подключить `post-hq` к оркестратору `content-machine` как **третий формат доставки — «пост»** (наряду с рилс и карусель), и дотянуть хэндофф `ai-research → post-hq`, чтобы найденная идея/новость могла без ручного переноса уходить в написание текстового поста.

Сегодня хэндофф ручной: выход `ai-research` (идея/новость) — валидное сырьё, но `content-machine` про текстовый пост не знает и в производство гонит только `reels-script`/`carousel-script`. Этот цикл закрывает разрыв.

## Принятые решения (развилки брейншторма)

1. **Пост = третий формат в `content-machine`** (тонкий оркестратор), а НЕ отдельный маршрут в обход машины. Единая дверь, DRY.
2. **«Пост» = самостоятельный текстовый артефакт** (Telegram / Threads / X / LinkedIn). Instagram-подпись остаётся внутри `post-hq` (`instagram-caption`) как компаньон визуала и из `content-machine` как формат НЕ дёргается. Чёткая граница: пост = текст-первичный артефакт.
3. **Площадку × цель разруливает `post-hq` сам.** `content-machine` их НЕ выводит и НЕ накладывает голос/цель — только маршрутизирует и пробрасывает сырьё.

**Отвергнутая альтернатива:** «толстая» машина, которая знает площадки и авто-бандлит IG-подпись к визуальным форматам — противоречит DRY-принципу `content-machine` (она только распознаёт/маршрутизирует/делегирует).

## Границы (что меняем / что НЕ трогаем)

| Скилл | Изменение |
|---|---|
| `content-machine` | + формат «пост» в рекомендации; + делегат `post-hq` в производстве/хэндоффе; чекпойнты и headless распространяются на пост; строка в «Где что брать». |
| `ai-research` | Дотянуть хэндофф: `ideas` → += `post-hq`; `digest` → добавить строку хэндоффа в `post-hq` (новость как сырьё для поста-комментария). |
| `post-hq` | Функционально НЕ меняется (уже принимает сырьё, сам разруливает площадку×цель×review). Опц. одна строка в `README.md` про вызов из `content-machine`. |
| `carousel-script`, `reels-script`, `instagram-caption` | Без изменений. |

## Изменения по скиллам

### content-machine (4 точки)

1. **Рекомендация формата** (`SKILL.md`, раздел «Рекомендация формата») — добавить четвёртый вариант:
   - **пост** (`post-hq`) — идея держится на тексте: тейк/мнение, новость + комментарий, анонс, личная история, разбор словами без обязательного видео или слайдов.
   - Существующие эвристики сохраняются дословно (рилс = одна эмоция/хук/динамика; карусель = пошаговый разбор/список/структура/сравнение/чек-лист), чтобы «пост» не перетягивал визуальные форматы. Критерий разведения: нужен ли визуал (видео → рилс; слайды → карусель) или идея самодостаточна как текст (→ пост).
   - Если пользователь задал формат явно — берётся его выбор, рекомендация не навязывается (как сейчас).

2. **Производство и хэндофф** (`SKILL.md`, раздел «Производство и хэндофф») — добавить цель производства:
   - **пост** → `post-hq`. Проброс: суть идеи + ниша + (если задано драйвером) площадка — как есть. Площадку/цель машина НЕ выводит; это работа `post-hq`.
   - «оба»/кросс-формат остаётся как есть (рилс+карусель из одного зерна); комбо с постом не вводим в этом цикле (YAGNI) — пост выбирается как самостоятельный формат.

3. **Чекпойнты** (`SKILL.md`, раздел «Чекпойнты») — новый тип НЕ нужен:
   - `research` / `reference-niche`: на чекпойнте пользователь может выбрать у кандидата формат «пост» → идея уходит в `post-hq`.
   - `own-idea` / `reference-single`: без чекпойнта, сразу к рекомендованному формату (включая пост).

4. **Headless (Hermes)** (`SKILL.md`, раздел «Headless-режим») — формат «пост» наследует headless:
   - Если входная инструкция задаёт источник + конкретную идею + формат (+ опц. площадку) — без вопросов, проброс в `post-hq` пасс-тру. Машина площадку НЕ парсит, передаёт строкой как есть; `post-hq` сам её прочитает (а в отсутствие — применит свою рекомендацию/дефолт).
   - Плюс строка в разделе «Где что брать (делегаты)»: *Производство поста — `post-hq`*.

### ai-research (хэндофф)

- Режим `ideas` (строка «Хэндофф»): к целям добавить **`post-hq`** (идея-тезис → текстовый пост) рядом с `reels-script`/`carousel-script`.
- Режим `digest`: добавить строку хэндоффа (сейчас отсутствует): *любую новость дайджеста можно передать в `post-hq` как сырьё для поста-комментария*. Прямо обслуживает «нашёл новость → написал пост».
- Режим `weekly`: без изменений (хэндофф в `carousel-script`, день = слайд).

### post-hq

- Функционально без изменений. Опционально — одна строка в `skills/post-hq/README.md`: что штаб вызывается в т.ч. из `content-machine` как формат «пост» (вход — сырьё; площадку×цель разруливает сам).

### Документация

- `skills/README.md` (каталог-витрина) и `skills/content-machine/README.md` — в описании делегатов `content-machine` добавить `post-hq` (третий формат).
- По завершении цикла — отметить в роадмапе `docs/superpowers/specs/2026-06-23-post-hq-design.md` (пункт про интеграцию в content-machine), что авто-связка сделана.

## Контракт делегирования (резюме)

```
content-machine  --(суть идеи + ниша + опц. площадка, как есть)-->  post-hq
                                                                       |
                          post-hq: лестница вмешательства → площадка×цель
                          → слои (каркас → чистка → голос) → review до 🟢
                                                                       |
                                                          готовый текст поста
```

`content-machine` не знает о площадках, целях, голосе, фреймворках и review — всё это единственный источник правды `post-hq`. Контракт — «сырьё на вход, готовый пост на выход».

## Проверка (поведенческий eval, свежий контекст)

1. **Тейк/мнение** — «сделай контент: AI усиливает, а не заменяет» → машина рекомендует **пост** → `post-hq`.
2. **Research → пост** — «найди тему про AI и напиши пост» → `ai-research` (`ideas`) → чекпойнт → формат «пост» → `post-hq`.
3. **Граница визуала** — «разбор по шагам, как внедрить AI» → всё ещё **карусель** (пост не перетягивает структурный разбор).
4. **Граница визуала-2** — «карусель для инсты про X» → `carousel-script` (не пост; визуальные форматы не уходят в post-hq).
5. **Headless** — полная инструкция (источник + идея + формат=пост + площадка) → без вопросов, проброс в `post-hq`.
6. **Digest → пост** — «дайджест AI за неделю», затем «из новости №2 сделай пост» → новость как сырьё уходит в `post-hq`.

Расхождения (чаще — над-/недо-срабатывание рекомендации «пост») фиксируются и правятся, упавшие кейсы перепрогоняются. Затем — гейт автора.

## Вне scope (отдельные будущие циклы)

- **Рантайм/запуск (Hermes-слой):** где и как реально гоняется конвейер, как с ним общаться. Отдельный брейншторм.
- **Источники ai-research:** откуда тянуть инфу (доп. скиллы / MCP; Perplexity уже в source-лесенке, докрутить охват). Отдельный брейншторм.
- Комбо «пост + рилс/карусель» из одного зерна — не вводим сейчас (YAGNI).
