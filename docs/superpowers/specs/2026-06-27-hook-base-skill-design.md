# Скилл `hook-base` — дизайн

**Дата:** 2026-06-27
**Статус:** утверждён (брейншторм)
**Маршрут дальше:** writing-plans → skill-creator (упаковка)

## Проблема

Контентным скиллам (`reels-script`, `carousel-script`) нужны сильные хуки, но сейчас они выдумывают их «с головы» через встроенный хребет виральности. Оба скилла уже содержат контракт «Порядок разрешения источника хука/обложки», где **шаг 2 = внешняя база паттернов, сегодня пуст**. Автор хочет:

1. Находить более сильные хуки **быстрее**, не каждый раз выдумывая.
2. **Быстро добавлять** клёвые хуки словами («залетел ролик с таким хуком — внеси», «увидел хук „…“, добавь»), без скрапера и ручного анализа.

Скрапер (`instagram-analytics`) — лишь один из источников, не единственный.

## Решение (одной фразой)

Один скилл `hook-base` = **библиотека + резолвер**: курируемый склад хук-паттернов по единой схеме плюс логика двух операций — **ПОЛОЖИТЬ** (приём хука с молчаливой классификацией) и **ВЗЯТЬ** (резолв: выбрать и адаптировать под тему). Без суб-агентов; доступ — обычный вызов скилла.

## Решения по развилкам (утверждено)

| Развилка | Решение |
|---|---|
| Роль | Библиотека + резолвер (не генератор, не чистый склад) |
| Доступ | Вариант Б: тот же агент вызывает скилл `hook-base`. Суб-агенты НЕ городим (даже на вырост) |
| Хранилище | Каноническое — Obsidian-волт (принцип «единственный склад»). В репо — только схема + логика + 1 обезличенный пример |
| Приём | Молча классифицирует; ответ одной строкой; спрашивает только при реально неясной нише/смысле |
| CTA | База хранит **только хуки**. CTA остаётся одной константой в `content-studio/CLAUDE.md` — никакой вариативности, чтобы не сбивать автора |

## Архитектура

```
   ТЫ словами ───▶ ПОЛОЖИТЬ (приём) ── классифицирует молча → .md
                                 │
                                 ▼
                   <волт>/2-Контент/База хуков/   ← склад .md по схеме
                                 ▲
 reels/carousel ──▶ ВЗЯТЬ (резолв, шаг 2) ── выбрать+адаптировать → 2–3 хука
 (или ты сам)
```

### Компоненты скилла (репо `skills/hook-base/`)

- `SKILL.md` — диспетчер: определяет операцию (положить / взять), несёт логику обеих.
- `references/schema.md` — схема записи, правила молчаливой классификации, конвенция имён, путь к складу в волте.
- `references/patterns/` — 1 обезличенный пример-затравка (для тех, кто заберёт харнес; у них склад свой, пустой).
- `README.md` (рядом) + строка в `skills/README.md` — витрина.
- `evals/evals.json` — тесты триггера и поведения.

### Склад (Obsidian-волт)

- Место: `2-Контент/База хуков/` — отдельно от `Разведка/` (сырой вывод скрапера). Лучший хук из `Разведка` **повышается** в `База хуков` как чистая запись.
- Путь волта скилл берёт из конвенции (прописан в глобальном `~/.claude/CLAUDE.md`). На других машинах путь параметризуется.
- `База хуков.base` — Obsidian-вид для человека (листать по тип_хука / ниша / метрикам). Резолвер вид не «исполняет» — он сканирует фронтматтер файлов.

## Схема записи

Расширяет существующую схему (`reels-script/references/patterns/README.md`): добавлены `формат` и `шаблон`.

```yaml
---
тип: hook
тип_хука: отрицание        # провокация | неожиданность | интрига | отрицание | обещание
ниша: ИИ
формат: рилс               # рилс | карусель | универсальный
шаблон: "{X} не заберёт {Y} — заберёт тот, кто умеет с этим работать"
когда_работает: "страх потери работы из-за ИИ"
кросс_нишевость: да        # да | нет — переносим ли в другую нишу
источник: "ручной"         # URL | "залетел, 120k просмотров" | "ручной"
метрики: { просмотры: null, охват: null }
---
ИИ не заберёт твою работу — её заберёт тот, кто умеет с ним работать
```

- `шаблон` — обобщённый скелет (резолвер подставит тему); тело файла — живой пример.
- Имя файла: `hook-<типхука-translit>-<ниша-translit>-NN.md`, напр. `hook-otritsanie-ai-01.md`. Кириллица — внутри фронтматтера.
- `NN` — следующий свободный номер для пары тип_хука+ниша.

## Операция ПОЛОЖИТЬ (приём)

Триггеры: «внеси хук», «добавь хук», «залетел ролик с хуком…», «запомни этот хук».

1. На входе — текст хука (опц.: просмотры/охват, ссылка, явная ниша).
2. Скилл **молча** выводит: `тип_хука`, `ниша` (дефолт — ниша автора, ИИ, если явно не иное), `формат`, `когда_работает`, `кросс_нишевость`, `шаблон` (обобщает конкретный хук). `источник` = что назвал автор. `метрики` — если названы.
3. Пишет `.md` в `2-Контент/База хуков/` с нужным `NN`.
4. Отвечает **одной строкой**: `Записал: отрицание · ИИ · рилс → hook-otritsanie-ai-02.md`.
5. Спрашивает ТОЛЬКО при реально неясной нише/смысле.

## Операция ВЗЯТЬ (резолв)

Вход: тема/идея, ниша, стадия (охватный/прогревочный), формат (рилс/карусель), желаемый тип хука (опц.).

1. Сканирует фронтматтер записей в `База хуков`.
2. Фильтр: `формат` совпадает (или `универсальный`); `ниша` совпадает ИЛИ (`кросс_нишевость: да` и переносимо); `тип_хука` если запрошен.
3. Ранг: своя ниша > кросс-ниша; есть метрики > нет; совпадение `когда_работает` с углом/болью.
4. Берёт топ-2–3, **подставляет `шаблон` под конкретную тему**, помечает дефолт.
5. Пусто по теме → честно сообщает и отдаёт управление обратно (вызывающий падает на встроенный хребет).

## Проводка в контентные скиллы

- `reels-script` → «Порядок разрешения источника хука», шаг 2: с «внешняя база не подключена» на «вызови `hook-base` (взять) с (ниша, стадия, формат=рилс, угол); возьми кандидатов; пусто → шаг 3 (встроенный хребет)».
- `carousel-script` → «Порядок разрешения источника обложки-хука», шаг 2: аналогично, формат=карусель.
- Встроенный хребет виральности остаётся фоллбэком.
- **CTA-логику не трогаем** (остаётся константой).

## Тесты (`evals/evals.json`)

- **Триггер:** (а) «внеси хук „…“» → ПОЛОЖИТЬ; (б) «дай хук под рилс про X» → ВЗЯТЬ; (в) не путать с `reels-script` / `instagram-analytics`.
- **Поведение:** запись соответствует схеме (все поля проставлены); резолв возвращает 2–3 хука, адаптированных под тему, с пометкой дефолта; при отсутствии совпадений — честный фоллбэк.

## Вне рамок (YAGNI)

- Суб-агент-доставщик (вариант А) — на вырост, без переписывания склада (схема та же).
- Отдельный машиночитаемый индекс/каталог склада — пока база мала, резолвер сканирует фронтматтер; добавим при росте.
- Хранение CTA-вариантов, скриптов целиком (`тип: script`) — позже, если понадобится.
