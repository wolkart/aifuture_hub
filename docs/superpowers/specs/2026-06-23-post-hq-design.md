# Штаб `post-hq` — дизайн

> Статус: спецификация, утверждена в брейншторме 2026-06-23.
> Назначение: штаб написания постов — оркестратор + роли-модули, который ведёт идею/черновик через голос автора → цель (контент/продажа) → площадку → ревью до готового поста. Многоплатформенный, на общем `brand-voice`.

## Контекст и цель

`post-hq` («штаб постов», HQ = headquarters = штаб) — НЕ один райтер, а штаб ролей. Принимает вход (идея / тезис / новость / черновик) и выдаёт готовый пост под нужную площадку и цель, в голосе автора. Строится НА уже готовом скилле `brand-voice` (единый источник голоса).

**Ключевое различение.** «Воронка» бывает (1) автоматизация-рантайм (многошаговые касания, триггеры — слой Hermes/n8n, вне этого скилла) и (2) контент — продающие/прогревочные **посты**. `post-hq` закрывает (2): продающий пост и экспертный пост — одна работа («написать пост»), разница лишь в **линзе убеждения**. Поэтому это режим внутри одного штаба, а не отдельный скилл.

### Решение «один штаб, не несколько скиллов» (аргументация)
1. Один фундамент: `brand-voice` + `writing-principles` применяются к любому посту. Разделение = дублирование подключения голоса.
2. Продающий пост — это режим (другой фреймворк-линза), не отдельная машина.
3. Референс (ogilvy-content-machine Владилена Минина) доказывает модель «оркестратор + модули платформ + общий voice»; мы добавляем измерение цели и слой review.
4. Один вход «напиши пост» → один штаб разруливает. Два скилла = конфликт триггеров.
5. Знание (Хормози/Огилви) живёт как референс; переиспользуется будущей воронкой-автоматизацией.

## Архитектура: два независимых измерения + слои

Пост = **платформа × цель**. Комбинируем модули (4 платформы + 2 цели = 6 файлов, не 8).

```
skills/post-hq/
├── SKILL.md                       ← оркестратор штаба
├── references/
│   ├── platforms/                 ← КАК (формат/структура площадки)
│   │   ├── telegram.md            ← Фаза 1
│   │   ├── threads-x.md           ← Фаза 2
│   │   ├── linkedin.md            ← Фаза 2
│   │   └── instagram-caption.md   ← Фаза 2
│   ├── purposes/                  ← ЗАЧЕМ (линза убеждения)
│   │   ├── informational.md       ← экспертный/контентный (Огилви-лайт)
│   │   └── selling.md             ← продающий/прогревочный (Хормози + Огилви)
│   ├── frameworks/                ← дистиллированное знание (чек-листы приёмов)
│   │   ├── hormozi.md
│   │   └── ogilvy.md
│   └── review.md                  ← роль-ревьюер (чек-лист прохода)
└── README.md
```

- **Голос НЕ дублируется** — оркестратор читает `skills/brand-voice/references/brand-voice.md` + `writing-principles.md` как единый источник.
- **Модули = роли-инструкции** (стиль «ты — агент…», как у Минина): один Клод по очереди исполняет роли (писатель → ревьюер) в одном контексте. Реальные dispatched-агенты — забота Hermes-рантайма позже; дизайн совместим (каждый модуль самодостаточен).

## Применение фреймворков (best practice)

Файлы-источники `AI-Nastavnik-Hormozi-100M-Offers.md` и `Инструкция_AI-наставник_Огилви.md` (~60КБ каждый) — это полноценные «AI-наставники» (роль + методология + кейсы), сделаны под диалогового наставника, не под написание постов. Интеграция по принципам:

1. **Дистилляция, не дамп.** Из каждого наставника извлекается компактный operating-чек-лист (приёмы «к применению» при письме поста): Хормози — Value Equation, Grand Slam Offer (5 шагов), усилители, гарантии, scarcity/urgency, MAGIC; Огилви — тест большой идеи, правила заголовка, body-copy, ясность. Персона-наставник, диалог-гайд, кейсы, отчёт верификации — отбрасываются.
2. **Условная загрузка (progressive disclosure).** `selling.md` грузит `hormozi.md` + `ogilvy.md`; `informational.md` — только `ogilvy.md` (облегчённо). Хормози к обычным постам не подмешивается.
3. **Голос — последним фильтром.** Стек: `brand-voice` → `writing-principles` → фреймворк цели (скелет убеждения). Голос накладывается последним: переозвучивает в стиль автора и срезает инфоцыганщину (`brand-voice §9`). Скелет чужой — кожа автора.
4. **Знание ≠ поведение.** `frameworks/*.md` — знание; `purposes/*.md` — роль, которая его применяет. Огилви шарится (selling + informational) → отдельный файл (DRY). Хормози — тоже отдельный (симметрия, переиспользование).

**Почему так:** раздутый контекст роняет качество и цену (lost-in-the-middle, размытие внимания); узкий релевантный держит модель острой — это и есть модель Agent Skills (progressive disclosure). Дистилляция верна источнику (приёмы — методы/идеи, не копирайт-выражение), формулируется своими словами.

**Копирайт/приватность:** полные 60КБ-наставники (производное от книг Хормози/Огилви) — ТОЛЬКО локально (`docs/superpowers/working/`, gitignore). В публичный скилл едет лишь наш дистиллят-чек-лист (наши директивы, без дословного текста книг). Аналогично руководству Ильяхова в `brand-voice`.

## Оркестратор (`SKILL.md`)

Frontmatter: `name: post-hq` + `description` с триггерами («напиши пост», «продающий пост», «прогрев», «пост в телегу / threads / linkedin», «оформи в моём голосе»).

Рабочий цикл:
1. **Вход** — идея / тезис / новость / черновик + (опц.) площадка и цель.
2. **Шаг 0** — читает `brand-voice` (голос) + `writing-principles` (чистота).
3. **Роутинг** — определяет платформу × цель. Сказал пользователь — берёт его выбор; не сказал — **рекомендует** (как Минин: «это лучше зайдёт как продающий для Threads + карточки»).
4. **Генерация (роль писателя)** — читает `platforms/<X>.md` + `purposes/<Y>.md` (+ `frameworks/*` если selling), пишет.
5. **Review-цикл (роль ревьюера)** — прогон по `review.md`; брак → конкретные замечания назад писателю; ок → выдача.
6. **Карточки (Threads/IG)** — при необходимости делегирует `carousel-script` (не дублирует визуал).
7. Границы: НЕ постит и не доставляет (слой рантайма); не переписывает авторский замысел (наследует лестницу вмешательства из `brand-voice`/`SKILL.md`).

## Контракт модуля платформы

Каждый `platforms/<X>.md` по единому шаблону (роль-инструкция): **роль · длина и плотность (под формат!) · структура (хук → тело → концовка) · вёрстка площадки · типы CTA · чек-лист · антипаттерны**. Фаза 1 — `telegram.md` (лонгрид; 2-строчная вёрстка и «длинное тире по необходимости» — из `brand-voice §8`). Фаза 2 — `threads-x.md`, `linkedin.md`, `instagram-caption.md` по тому же контракту.

**Длина/плотность — ОБЯЗАТЕЛЬНО под формат.** Каждый модуль задаёт свою длину по принципу «тип поста + плотность ценности, не число абзацев»: новость/тейк/анонс → коротко; глубокий разбор → можно длиннее, если каждый блок несёт ценность и абзацы воздушные; критерий — «можно убрать предложение без потери → убрать». Threads/X — заведомо короткий формат (плотные лонгриды не для него); LinkedIn/IG-подпись — свои лимиты. Абзац везде короткий и воздушный.

## Модули цели

- `purposes/informational.md` — экспертный/контентный: полезное действие, миф → разбор, числа-якоря (Огилви-лайт + `writing-principles`).
- `purposes/selling.md` — продающий/прогревочный: оффер, боль → решение, ценность/усилители/гарантии/дедлайн (Хормози + Огилви), но через голос автора и без инфоцыганщины (страховка `brand-voice §9`).

## Review-слой (`review.md`)

Роль ревьюера проверяет текст писателя по: (а) светофору `brand-voice §6` (🟢/🟡/🔴); (б) анти-шаблону (`§4`); (в) вёрстке/длинному тире (`§8`); (г) чек-листу платформы; (д) для selling — честность оффера (не обещать лишнего). Брак → конкретные замечания назад писателю, цикл повторяется. Это «review-it агент» как модуль-инструкция.

## Композиция и границы

- **brand-voice** — фундамент, читается всегда первым (единый голос на всех площадках и целях).
- **carousel-script** — делегат визуала (пост + карточки).
- **content-machine** — сосед; позже сможет роутить текст-посты в `post-hq` (НЕ в этом цикле, YAGNI).
- **Hermes** — рантайм; позже поднимет модули-роли как реальных dispatched-агентов (дизайн совместим).

## Фазы

- **Фаза 1 (этот цикл):** `SKILL.md` (оркестратор) + подключение `brand-voice` + `platforms/telegram.md` + обе `purposes/` + `frameworks/hormozi.md` + `frameworks/ogilvy.md` + `review.md` + `README.md` + каталог + install. Полностью рабочий штаб под Telegram (контент и продажа).
- **Фаза 2 (следующий цикл):** `platforms/threads-x.md`, `linkedin.md`, `instagram-caption.md` по контракту платформы.

## Входы (есть на диске, локально)
- Голос: `skills/brand-voice/` (готов).
- Наставники-источники: `docs/superpowers/working/AI-Nastavnik-Hormozi-100M-Offers.md`, `Инструкция_AI-наставник_Огилви.md`.
- Референс (форма модулей-ролей): ogilvy-content-machine Минина (`telegram.md`, `threads.md` — изучены).

## Вне области (не делаем)
- Постинг/доставка контента (рантайм/Hermes).
- Воронка-автоматизация (многошаговые последовательности, триггеры).
- Интеграция в `content-machine` (позже) — **роадмап:** `content-machine` (роутер источников) передаёт найденную `ai-research` новость/идею в `post-hq` как ещё один формат доставки (наряду с reels/carousel). Сегодня хендофф ручной (выход ai-research = валидное «сырьё» для post-hq); авто-связку делаем отдельным циклом после Фазы 2.
- Реальные dispatched-агенты внутри скилла (это слой Hermes).
- Фаза 2 платформы (отдельный цикл).
