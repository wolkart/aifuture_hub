# Скилл `strategist` — дизайн

**Дата:** 2026-06-30
**Статус:** утверждён (брейншторм)
**Маршрут дальше:** writing-plans → meta-prompt (докрутка ядра под 2 режима) → skill-creator (упаковка, evals)

## Проблема

Контент и решения автора плывут в абстракцию: нет инструмента, который на ходу вправляет позиционирование/аватар и вытаскивает конкретику. Есть проверенный промпт «Стратегический Интервьюер» (в волте `4-Ресурсы/База промптов/`), которым автор в Claude web через Project прошёл мощное интервью и вышел на своё позиционирование (strategic_map). Но он заперт как одноразовый веб-проект — нельзя переиспользовать, нельзя быстро «проверить на дрейф».

## Решение

Публичный hub-скилл `strategist` — обезличенный стратег-коуч по позиционированию (Траут/Райс, категорийный дизайн, JTBD, Value Proposition Canvas, Хормози). Ядро = существующий промпт, очищенный от личного контекста. Два режима.

## Решения по развилкам (утверждено)

| Развилка | Решение |
|---|---|
| Природа | **Публичный генерик** (как meta-prompt). Переиспользуемая методология; личные ответы/карта автора — в волте, НЕ в скилле |
| Режимы | **Два:** полное ИНТЕРВЬЮ + лёгкий ДРЕЙФ-ЧЕК |
| Позиционирование пользователя | **Вход, не зашивка:** для дрейф-чека пользователь даёт свою карту/CTA или указывает файл |
| Связь с харнесом | **Отдельный, on-demand.** Дрейф-чек как ревью-проход над контентом — опционально и позже; в контентные скиллы сейчас НЕ вшиваем |

## Архитектура

- `SKILL.md` — роль + диспетчер режимов + дисциплина ответа.
- `references/interview-method.md` — 4 фазы интервью (обезличенные из исходного промпта).
- `references/drift-check.md` — рубрика дрейф-чека.
- `README.md` + строка в `skills/README.md`.
- `evals/evals.json`.

### Режим 1 — ИНТЕРВЬЮ

Полные 4 фазы (из исходного промпта):
1. **Рентген** — идентичность, уникальный опыт, что бесит, за что благодарят.
2. **Вскрытие продукта** — что реально покупают (JTBD, Value Proposition Canvas как внутренняя навигация).
3. **Позиционирование и монетизация** — аватар хирургически, враг, категория, ценовая логика, кто НЕ клиент.
4. **Сборка** — сам собирает стратегическую карту (6 разделов: миссия / УТП / аватар-психография / продуктовая линейка / позиционирование / слепые зоны+рекомендации).

Принципы: один вопрос за раз; не принимать первый ответ; не подсказывать; фиксировать противоречия; маркировать неуверенность; не генерировать факты; при абстракциях возвращать к конкретике. Промежуточная фиксация в конце каждой фазы. Финал — **артефакт-карта**; пользователь сам сохраняет (скилл НЕ перезатирает существующую карту).

### Режим 2 — ДРЕЙФ-ЧЕК

Вход: решение/контент + позиционирование пользователя (карта/CTA/аватар). Скилл хирургически указывает, **где дрейф**: абстракция вместо конкретики; мимо аватара; мимо врага/обещания; продажа вместо ценности; размытый CTA. Даёт конкретный возврат к позиционированию. Коротко, без интервью. Это «наставник, который бьёт по рукам».

Рубрика дрейф-чека (`references/drift-check.md`): аватар · враг · обещание · конкретность (есть ли цифра/точка Б) · «продажа vs ценность» · CTA (кодовое слово + конкретное обещание, не абстракция).

## Определение режима

- «проведи интервью / собери позиционирование / стратегическую карту с нуля» → ИНТЕРВЬЮ.
- «проверь на дрейф / это в моём позиционировании? / где я сплыл» + даёт материал → ДРЕЙФ-ЧЕК.

## Дисциплина ответа

Без думания вслух. ИНТЕРВЬЮ — один вопрос за раз, никаких списков. ДРЕЙФ-ЧЕК — сжатый разбор «где дрейф → возврат», без преамбул.

## Тесты (evals)

- Триггер: (а) «проведи стратегическое интервью» → ИНТЕРВЬЮ; (б) «проверь этот пост против моего позиционирования» → ДРЕЙФ-ЧЕК; (в) не путать с meta-prompt (тот про промпты) / post-hq (тот пишет пост).
- Поведение: интервью задаёт по одному вопросу и не принимает первый ответ; дрейф-чек называет конкретные точки дрейфа + возврат; факты не выдумываются.

## Вне рамок (YAGNI)

- Автоматический ревьюер-слой в контентном пайплайне (позже, опц., по практике).
- Авто-обновление документов в волте (скилл отдаёт артефакт, пользователь сохраняет сам).
- Зашивка личного контекста автора в скилл (он генерик).
