# Дизайн скилла `carousel-script` (карусель-продюсер)

Дата: 2026-06-18
Статус: утверждён, готов к плану реализации

## 1. Цель

Скилл превращает идею/тезис в готовую **карусель** (пост-карусель) для Instagram и/или LinkedIn: цепляющая обложка, одна ключевая мысль, раскладка по слайдам с атомизацией, плюс **per-slide спек**, по которому слайды можно отрендерить (Figma-MCP / image-gen) или собрать вручную.

Результат — **текст карусели + per-slide спек** (артефакт). Скилл самодостаточен сегодня (выдаёт спек для ручной сборки) и спроектирован так, чтобы авто-рендер подключался без переписывания.

## 2. Границы (scope)

**Входит:** сам скилл `carousel-script` (контент + спек).
**НЕ входит:** реальный рендер слайдов (это шаг тула Figma-MCP / image-gen в рантайме), оркестратор-«content-machine», постинг. Скилл только закладывает чистый **render-agnostic контракт** (per-slide спек), к которому рендер подключится позже.

## 3. Личность и «штаб»

`carousel-script`. `SKILL.md` = оркестратор (флоу). `references/instagram.md` + `references/linkedin.md` = роли-субагенты под платформы. **Платформа = режим**, хребет виральности один.

- IG / LinkedIn / **обе**. При «обе» оркестратор **спавнит субагента на каждый платформенный бриф** → две заточенные версии (параллельно).
- Уточнять платформу у пользователя только если критично; иначе вывести и зафиксировать в «Допущениях».

## 4. Флоу

1. **Гейт ширины** — «Интересно ли это 5+ млн?». Узко/жаргон → расширить (см. хребет). Если тема душит — сказать прямо и предложить шире.
2. **Одна ключевая мысль** — одним предложением.
3. **Платформа + угол** — определить (IG / LinkedIn / обе).
4. **Обложка (хук-заголовок)** — см. §6; источник по «порядку разрешения» (как у reels): пользователь → внешняя база (пока не подключена) → встроенный хребет.
5. **Раскладка по слайдам** — структура §5, каждый слайд атомизирован (наблюдаемое действие = слайд).
6. **Самопроверка** (тесты §8) → провален — переделать.

## 5. Структура слайдов

- **Обложка-заголовок** (слайд 1) — останавливает свайп. Короткий цепляющий **заголовок** (часто 2–4 слова, иногда больше). Правило — **«помещается крупно на карточке»**, НЕ счётчик слов. Виральные принципы (провокация/любопытство/обещание) выражены коротким заголовком.
- **Слайды-ценность** (2…n-1) — **одна мысль на слайд**, атомизировано.
- **CTA-слайд** (последний) — призыв.

Объём: **IG 6–10** слайдов (короче, визуальнее), **LinkedIn 8–12** (плотнее, экспертнее).

## 6. Формат карточки

Вытянутый/портретный по умолчанию: **4:5 (1080×1350)** — параметр (можно менять). Влияет на то, сколько текста влезает.

## 7. Per-slide спек (центр, render-agnostic)

Каждый слайд описывается полями:
```
№ · роль(обложка/ценность/CTA) · заголовок · текст · подача(text-treatment) · лейаут · визуал-направление
```
- **подача (text-treatment)** — вместо лимита слов; отражает реальные стандарты:
  - `мега-текст` — на всю карточку огромными буквами (≈3 слова) + бренд-значок;
  - `заголовок+треть` — заголовок + текст ~1/3 карточки;
  - `плотный-текст` — компактно, слов больше (зависит от темы).
- **лейаут (словарь)** — `большая-цифра · цитата · чеклист · текст+визуал · сравнение` (+ обложка). Стабильный словарь → потом мапится 1:1 на готовые Figma-компоненты пользователя.
- **визуал-направление** — что показать; **эта же строка = промпт для image-gen**.

Правило слайда: текст **помещается на вытянутой карточке** при выбранной подаче.

Плюс **метастрока** на всю карусель: `ниша · платформа · угол · тип хука`.

## 8. Тесты перед выдачей

Ширина (5М+) · одна мысль на слайд · обложка останавливает свайп · атомизация (наглядность) · CTA на месте · текст влезает на карточку. Провален — переделать часть.

## 9. Рендер-лесенка (opt-in, ручной override главнее)

- Скилл **всегда выдаёт спек** (ручная сборка работает всегда).
- Авто-рендер запускается **только по явному запросу** пользователя: **Figma-MCP → image-gen → (иначе) руками**.
- **Ручной режим — приоритетный override:** если пользователь сказал «я сам» (или это дефолт) — авто-рендер не запускается, **даже если Figma подключена**.
- Дефолт — ручная сборка.

## 10. Формат вывода

1. **Слайды по битам** — каждый слайд с ролью и контентом.
2. **Per-slide спек** (§7) — таблицей или блоком, готов к рендеру/ручной сборке.
3. **Метастрока** — `ниша · платформа · угол · тип хука`.
4. При «обе платформы» — две версии (IG и LinkedIn).
5. После — блок **«Допущения»**.

Дисциплина ответа — как у reels: оценку/гейт/тесты делать молча, без «думания вслух», без финального CTA к диалогу.

## 11. Файлы

```
skills/carousel-script/
├── SKILL.md                  # роль, штаб, флоу, порядок источника хука, дисциплина, рендер-лесенка
├── README.md                 # где применять + .env.template-пункт под Figma/image-MCP
└── references/
    ├── carousel-backbone.md  # свой хребет под свайп-механику (обложка, свайп-удержание, атомизация, расширение темы, CTA)
    ├── instagram.md          # IG-стандарты подачи (text-treatments), тон, объём
    ├── linkedin.md           # LinkedIn: плотнее, экспертно, B2B
    └── slide-spec.md         # схема per-slide спека + словарь лейаутов и подач
```

Хребет — **свой** (само-достаточный скилл, качается отдельно); концепции те же, что у reels, но адаптированы под слайды/свайп.

## 12. Ограничения и переиспользование

- Наследуем из `reels-script`: адаптивные уточнения, дисциплина ответа, ниша=ИИ по умолчанию + кросс-перенос, стоп-лист клише, тон-регулятор, конкретика вместо абстракций.
- Ядро инструкции — через скилл `meta-prompt` (правило рекурсии в `CLAUDE.md`), упаковка через `skill-creator`.
- После сборки — подключить глобально скриптом `skills/install-skills.*` и добавить в витрину `skills/README.md`.

## 13. Non-goals (YAGNI)

- Не пишем код рендера в скилле (рендер — внешний тул).
- Не строим оркестратор content-machine / постинг сейчас.
- Не делаем отдельные скиллы под IG/LinkedIn — пока режимы одного (граница прочерчена reference-ами, сплит потом тривиален).
- Не фиксируем точные возможности Figma-MCP — проверим при подключении; спек от них не зависит.
