# Дизайн скилла `reels-script` (РИЛС-продюсер)

Дата: 2026-06-17
Статус: утверждён, готов к плану реализации

## 1. Цель

Скилл превращает сырую идею/тезис в готовый **вирусный сценарий Reels/Shorts** с мощным хуком на 1–3 секунды, одной ключевой мыслью и широкой подачей (без техножаргона), бьющей в боли зрителя, который пришёл залипать.

Результат скилла — **текст сценария** (артефакт), а не выполнение задачи. Скилл работает самодостаточно уже сегодня (встроенный «хребет виральности») и спроектирован так, чтобы позже без переписывания подключить внешнюю базу вирусных хуков/сценариев.

## 2. Границы (scope)

**Входит в этот проект (хаб скиллов):** сам скилл `reels-script`.

**НЕ входит (отдельный проект, свой репозиторий):** база вирусных хуков/сценариев, выгрузка по ссылке на ролик, обвязка для подключения агента «Гермес» (VPS, Docker), выбор хранилища (Obsidian-vault vs векторная БД Pinecone/Supabase). Этот проект только закладывает **контракт**, к которому база подключится позже.

## 3. Две оси сценария

- **Угол** (*что* за сценарий): инструкция / разоблачение мифа / провокация / кейс-история / подборка.
- **Манера съёмки** (*как* снято): разговорный talking-head / новостной / личная история.

Гибрид: можно выбрать пресет или описать формат словами в моменте. **Хребет виральности одинаков для любой комбинации.**

## 4. Пошаговый флоу

1. **Гейт ширины** — «Интересно ли это 5+ млн человек?». Узко/технично → расширить, убрать жаргон; если тема душит аудиторию — прямо сказать и предложить более широкий угол.
2. **Одна ключевая мысль** — сформулировать одним предложением (Reels = 1 идея, не 10).
3. **Угол + манера** — определить; уточнять у пользователя только если критично (адаптивное правило, как в `meta-prompt`).
4. **Хук** — порядок разрешения источника: (1) хук пользователя → (2) внешняя база, если подключена → (3) встроенный хребет. Если хук дан — лёгкая критика с высоким порогом (возражать только на явный мискаст, **с конкретной причиной**; спорный/необычный — пропускать). Если не дан — сгенерировать 2–3 варианта из хребта.
5. **Сборка сценария** по структуре (см. §5), с применением атомизации (§6).
6. **5 тестов** перед выдачей (см. §7); провален — переделать соответствующую часть.

## 5. Структура сценария (биты)

- **Хук** (1–3 сек, ≤10 слов) — остановить скролл; понятно без контекста; бьёт в эмоцию.
- **Усиление** (3–5 сек) — развернуть хук, конкретизировать проблему, намекнуть на решение.
- **Основной контент** (30–40 сек) — раскрыть пользу; структура зависит от угла (инструкция → шаги; разоблачение → проблема/почему/что вместо; кейс → было/стало/как; подборка → топ-N).
- **Вывод** (5–7 сек) — закрепить главную мысль.
- **Лидмагнит** (всегда) — призыв; формулировка адаптируется под формат: мягкий CTA (коммент/подписка) для разговорного, полноценный лидмагнит с кодовым словом для ценностного.

Ограничение длины: ~200–300 слов (≈60 сек озвучки).

## 6. Хребет виральности (`references/viral-backbone.md`)

Ось каждого сценария. Содержит:
- **Формулы хуков** (провокация / неожиданность / интрига / отрицание / обещание) + требования (≤10 слов, понятно без контекста, эмоция).
- **Атомизация (Хормози)** — раскладывать идею/совет до мельчайших **наблюдаемых действий-«атомов»**; каждый атом демонстрируем на экране (атом ↔ кадр).
- **Правило «абстрактное → наблюдаемое»** — не «используй ИИ правильно», а атомарные «добавь фразу X, потом сделай Y». Никаких невидимых качеств («будь экспертным») — только видимые действия.
- **Инверсия (опционально)** — генератор хуков для угла «разоблачение/провокация»: перечисли, что гарантированно проваливает, и переверни.
- **Удержание** — open loop, темп, ноль воды.
- **Расширение темы / уход от жаргона.**
- **CTA / лидмагнит-паттерны.**

Намеренно НЕ берём из источника Хормози: мотивационную рамку (инвестиции в себя, благодарность, «мозг ищет угрозы», терпеливость) — это контент его выступления, а не метод сценария.

## 7. Пять тестов перед выдачей

1. **Ширина** — интересно 5+ млн?
2. **Польза** — зритель получит конкретную ценность за ~60 сек?
3. **Провокация** — есть элемент, который заставит остановиться?
4. **Наглядность** — можно визуализировать каждую фразу (атомизация)?
5. **Лидмагнит** — логично вытекает из темы?

Любой провален → переделать соответствующую часть.

## 8. Формат вывода (DB-ready)

Основной вывод — сценарий по битам + строка «почему хук цепляет» + **метастрока в схеме паттерна** (см. §9): `ниша · угол · манера · тип хука`.

**Раскадровка** (таблица `Текст | Визуал`, кадр на атом, динамика, гипертрофия) — **опционально, по запросу** (не раздувает каждый ответ).

## 9. Инфраструктура под будущую базу (контракт, закладываем сейчас)

1. **Схема записи паттерна** — один хук/сценарий = запись с полями:
   `текст · тип хука · ниша · источник · метрики (просмотры/охват) · когда работает · кросс-нишевость (да/нет)`.
   Встроенные сид-хуки в `references/patterns/` лежат в этой схеме.
2. **Порядок разрешения источника хука** (зашит в `SKILL.md`): (1) пользователь → (2) внешняя база, если подключена → (3) встроенный хребет. Сегодня шаг (2) пуст: скилл отмечает «база не подключена» и идёт дальше. Завтра база пишет записи в той же схеме в известное место — логику скилла не трогаем.
3. **Выход = формат загрузки** — сценарий отдаётся с метаполями той же схемы → готовые записи для будущей базы («память роликов»). Цикл: сгенерил → сохранил → стало паттерном.

Встроенный «анализ конкурента → переупаковка (не копия)» = лайт-режим кросс-нишевого переноса хука (фитнес → ИИ), который позже усилит база.

```

└── references/
    ├── viral-backbone.md   # формулы хуков, атомизация, абстрактное→наблюдаемое,
    │                       # инверсия, удержание, расширение темы, CTA
    ├── formats.md          # углы × манеры съёмки, как комбинировать
    └── patterns/
        ├── README.md       # схема записи паттерна
        └── *.md            # сид-хуки в схеме (затравка будущей базы)
```

## 11. Ограничения

- Длина ~200–300 слов (≈60 сек); стоп-лист клише («в современном мире», «не секрет»…).
- Конкретика вместо абстракций (атомизация); узкие термины — только с объяснением.
- Тон/острота — регулятор (дефолт + смягчить/усилить), не зашитая агрессия.
- Ниша — параметр (дефолт ИИ, но с поддержкой кросс-нишевого переноса).
- Адаптивная дисциплина ответа: рассуждения молча, на выходе — сценарий, без «думания вслух».

## 12. Подход к сборке

Ядро инструкции пишем через скилл `meta-prompt` (роль, флоу, формулы), затем упаковываем и тестируем через `skill-creator`. Это соответствует правилу рекурсии в `CLAUDE.md` хаба.

## 13. Non-goals (YAGNI)

- Не строим саму базу данных и обвязку Гермеса в этом проекте.
- Не выбираем хранилище (Obsidian vs вектор) — решаем в отдельном цикле.
- Не парсим ссылки на ролики.
- Не добавляем мультиплатформенную генерацию (TikTok/YT-специфика) сверх манер съёмки — при необходимости позже отдельным reference.
