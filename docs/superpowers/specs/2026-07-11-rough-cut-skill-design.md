# Скилл `rough-cut` — дизайн

**Дата:** 2026-07-11
**Статус:** утверждён (брейншторм)
**Маршрут дальше:** writing-plans → реализация (скрипты + SKILL.md) → evals

## Проблема

Автор пишет видео раздельно: звук через OBS, видео с камеры (иногда параллельно — скринкаст OBS + камера). Черновой монтаж — рутина: синхронизировать дорожки по волне (как в CapCut/Resolve), вырезать тишину, филлеры («э-э», «пык-мык»), заикания и запоротые дубли («сказал криво — повторил»). В скринкастах тишина ≠ мусор: пока на экране что-то происходит (мышь, загрузка), участок нужен — но, возможно, ускоренный (таймлапс). Всё это сейчас делается руками в монтажке.

## Решение

Публичный hub-скилл `rough-cut` — «монтажёр черновой сборки». Берёт сырые записи, синхронизирует, находит мусор, для скринкастов смотрит на экран, и выдаёт результат в выбранном формате. Последнее слово — за пользователем: спорные решения помечены, правки итеративны.

## Решения по развилкам (утверждено)

| Развилка | Решение |
|---|---|
| Выход | **Cut-list JSON как канон** + сменные экспортёры: FCPXML (Resolve) и ffmpeg-рендер (mp4 — CapCut/рилсы/веб). Premiere XML — задел, отдельный экспортёр без изменения ядра. CapCut не импортирует EDL/FCPXML — для него только рендер |
| Дубли | **LLM по транскрипту** (Claude читает пословный транскрипт, находит оборванную фразу + повтор → оставить последний удачный). Без изменения привычек записи; спорное помечается |
| Синхрон | **Кросс-корреляция аудиодорожек** (камера пишет свой звук — подтверждено). Скрипт репортит confidence; при низкой — честно сказать, попросить хлопок/ручное смещение |
| Сценарии входа | Все три: камера+звук OBS; скринкаст OBS (один файл); скринкаст+камера одновременно (мультикам — одни резы на обе дорожки) |
| Скринкаст-тишина | **Авто + отчёт:** Claude смотрит кадры тихих участков, решает резать/таймлапс/оставить, выдаёт таблицу решений с причинами и уверенностью |
| Реализация | **Своё ядро** (вариант A): детерминированные Python-скрипты для механики, Claude — мозг для дублей/скринкастов. FCPXML auto-editor'а — как референс совместимости, не зависимость |
| Профили | **Два:** `лёгкий` (тишина+филлеры+дубли, ноль работы с кадрами — дёшево по токенам) и `полный` (+ визуальный анализ, таймлапсы). Определяется по входу автоматически; при скринкасте — один вопрос пользователю, если профиль не указан явно |
| Приватность | Обезличенный генерик (пути/пороги — параметры, ничего личного в скилле). Всё локально: whisper, ffmpeg — ничего не грузится наружу |

## Пайплайн (5 стадий)

1. **Intake** — `ffprobe` по входным файлам (разрешение, fps, дорожки); определение сценария и профиля по составу входа. Камера+звук → визуальный анализ не запускается вообще. Скринкаст → вопрос «смотреть на экран для таймлапсов или просто порезать по звуку?» (если профиль не задан при запуске).
2. **Sync** — кросс-корреляция звука камеры и OBS-дорожки → смещение в мс + confidence. Все дальнейшие таймкоды — в одной системе координат. Пропускается для одиночного скринкаста.
3. **Detect** — механика без LLM: тишина (`ffmpeg silencedetect`, пороги настраиваемые: минимальная длительность паузы, уровень dB, паддинг до/после речи); транскрипт с пословными таймкодами (`mlx-whisper`, локально).
4. **Decide** — LLM-часть (Claude в сессии): филлеры, заикания, дубли по транскрипту; для полного профиля — кадры из тихих участков (начало/середина/конец каждого) → статика = резать, движение/загрузка = таймлапс ×N, осмысленная короткая пауза = оставить. Каждое решение — причина + confidence; неуверенные помечены.
5. **Export** — по выбору из одного cut-list: FCPXML (резы «мягкие», исходники не тронуты; мультикам — две видеодорожки) и/или ffmpeg-рендер (H.264+AAC, таймлапсы ускоряются при рендере).

После пайплайна — **отчёт-таблица решений** (таймкод, длительность, что нашёл, что решил, уверенность). Правки («в 02:15 верни») = правка cut-list + переэкспорт, анализ заново не гоняется.

## Cut-list — канонический артефакт

`<имя>.cutlist.json` рядом с исходниками: входные файлы, смещение синхрона, профиль, сегменты `{start, end, action: keep|cut|speedup, rate, reason, confidence, source: silence|filler|retake|screencast}`. Единственный источник правды; человекочитаемый, правится руками.

## Архитектура

```
skills/rough-cut/
  SKILL.md            # роль, сценарии, профили, пайплайн, правила решений
  README.md           # описание для витрины (+ строка в skills/README.md)
  references/
    thresholds.md     # пороги тишины/паддинги, дефолты и как крутить
    decide-rules.md   # правила дублей/филлеров и скринкаст-решений
    fcpxml-notes.md   # шпаргалка совместимости FCPXML (Resolve)
  scripts/
    probe.py          # intake
    sync_offset.py    # кросс-корреляция → смещение + confidence
    detect_silence.py # ffmpeg silencedetect → интервалы
    transcribe.py     # обёртка mlx-whisper → пословные таймкоды
    extract_frames.py # кадры тихих участков (только полный профиль)
    export_fcpxml.py  # cut-list → FCPXML
    render.py         # cut-list → ffmpeg mp4 (с ускорением сегментов)
```

**Ключевой принцип (масштабирование):** вся логика — в скриптах и cut-list, НЕ в тексте SKILL.md. Скрипты не знают про Claude-сессию: вход — файлы/JSON, выход — JSON/артефакты. SKILL.md — только оркестрация и правила решений.

Зависимости: ffmpeg (есть), Python + `mlx-whisper`, `numpy/scipy`. Скилл проверяет наличие при запуске и подсказывает установку.

## Масштабирование (план на будущее, не в этой итерации)

Путь «личный скилл → продукт», возможный благодаря разделению ядро/интерфейс:

1. **Сейчас:** интерфейс = диалог с Claude, мозг = Claude в сессии.
2. **HTML-превью** (опция): страница с плеером + кликабельный список резов («тут вырезал 4 сек — послушать») для проверки без монтажки. Зародыш UI.
3. **Локальное веб-приложение:** скрипты оборачиваются в локальный сервер (FastAPI), превью становится редактором cut-list (плеер, галочки вернуть/вырезать, экспорт). LLM-стадия Decide переезжает в вызовы Claude Agent SDK.
4. **Десктоп:** тот же сервер пакуется в Tauri/Electron — «мини-CapCut для черновой резки».

Каждый шаг — смена обёртки, не переписывание ядра.

## Ошибки и фоллбэки

- Синхрон не сошёлся (низкая корреляция) → сказать прямо, предложить хлопок или ручное смещение.
- Whisper переврал → дубли ловятся и по структуре (пауза + лексический повтор); спорное помечено, не режется молча при низкой уверенности.
- FCPXML не открылся в Resolve → путь через ffmpeg-рендер всегда доступен.
- **Страж: исходники никогда не перезаписываются.** Все артефакты — новые файлы рядом.

## Тесты (evals)

- Триггер: «обработай запись / порежь тишину / синхронизируй звук и видео / сделай черновой монтаж» → rough-cut; не путать с instagram-downloader (тот качает чужое) и reels-script (тот пишет сценарий).
- Поведение: на фикстурах с заложенным «мусором» (тишина N сек, повтор фразы, филлер) cut-list ловит дефекты с корректными таймкодами; лёгкий профиль не вызывает extract_frames; FCPXML валиден (открывается в Resolve); рендер склеивает нужные куски и ускоряет speedup-сегменты; исходники не тронуты.
