# Дизайн скилла `instagram-downloader` (референс → переупаковка)

Дата: 2026-06-18
Статус: утверждён, готов к плану реализации (writing-plans)

## 1. Цель

Тул-скилл: по ссылке на Instagram-пост/рилс/карусель вытаскивает **медиа + подпись** (через RapidAPI) и передаёт их на **обязательную переупаковку** в `reels-script` / `carousel-script`.

**Ключевое (вшито в суть):** это **инструмент извлечения референса для создания СВОЕГО контента**, а НЕ качалка для репоста чужого. Переупаковка обязательна — скилл никогда не выдаёт «репостни как есть». Скачивается публичный/свой контент как референс; финальный артефакт — переупакованный оригинал.

## 2. Границы (scope)

**Входит:** извлечение ссылки → загрузка медиа + подписи → структурированный результат → **обязательный хэндофф на переупаковку**.
**НЕ входит:** репост чужого контента дословно (запрещено дизайном); доставка в Telegram (это слой рантайма/Hermes); аналитика профиля/метрики (отдельный скилл `instagram-analytics`); meta-prompt (скилл процедурный, не контентный).

## 3. Метод

**RapidAPI IG-скрейпер** (проверенный подход, конфигурируемый под подписку пользователя):
- `.env`: `RAPIDAPI_KEY`, `RAPIDAPI_HOST`.
- Вызов `https://<RAPIDAPI_HOST>/scraper?url=<INSTAGRAM_URL>` + заголовки `x-rapidapi-host`, `x-rapidapi-key`.
- Ответ: `{"data":[{"media":"<CDN_URL>","isVideo":true|false}, …]}` — массив = карусель из нескольких файлов; `media` — прямая CDN-ссылка; `isVideo` — тип.
- 💡 Рекомендуется выбрать RapidAPI-эндпоинт, который **возвращает и `caption`** — тогда подпись приходит в том же вызове (нужна для переупаковки). Если эндпоинт отдаёт только медиа — подпись добирается будущим `instagram-analytics` (Apify) или вводится вручную.
- **Опциональный бесплатный фоллбэк:** `yt-dlp` для публичного контента (без ключа), если RapidAPI не настроен. Парсинг ответа адаптируется под фактическую форму.

## 4. Флоу

1. **Извлечь ссылку** из сообщения: `instagram.com/p/…`, `instagram.com/reel/…`, `instagram.com/reels/…`.
2. **Запросить** через RapidAPI (или yt-dlp фоллбэк).
3. **Распарсить `data[]`**: скачать каждый media-файл в рабочую папку (карусель = несколько файлов), отметить `isVideo`.
4. **Извлечь подпись** (если эндпоинт её отдаёт).
5. **Выдать структурированный результат** (§5).
6. **ОБЯЗАТЕЛЬНО** — предложить/выполнить переупаковку: передать референс (подпись + тип + суть) в `reels-script` (для видео) или `carousel-script` (для слайдов/постов), с указанием ниши пользователя для кросс-переноса. Это финальная цель, не опция.

## 5. Выход (repackage-ready)

```
URL · тип (пост/рилс/карусель) · файлы (пути к скачанным медиа) · подпись · метрики (если есть)
```
Затем — **хэндофф-блок:** «Референс извлечён. Переупаковываю через `reels-script`/`carousel-script` под нишу <ниша>» (а не «вот медиа для репоста»).

## 6. Обязательная переупаковка / этика (вшито)

- Скилл **всегда** трактует загруженное как **референс для переупаковки**, не как готовый контент к репосту.
- Финальный артефакт — **переупакованный оригинал** через `reels-script`/`carousel-script` (у них уже есть режим «переупаковка чужого вирусного под свою нишу», кросс-перенос).
- Скилл не предлагает и не выполняет дословный репост.

## 7. Ограничения

- `.env`: `RAPIDAPI_KEY`, `RAPIDAPI_HOST` (плюс `.env.template`).
- Загрузка — инлайн через `curl` (RapidAPI) / `yt-dlp` (фоллбэк) в SKILL.md, без отдельного скрипта.
- Соблюдение ToS Instagram — на стороне пользователя; качаем публичный/свой контент как референс.
- Дисциплина ответа — как у всех скиллов хаба.

## 9. Non-goals (YAGNI)

- Не репостить чужой контент дословно (запрещено дизайном — только переупаковка).
- Не доставлять в Telegram (слой рантайма/Hermes).
- Не делать аналитику профиля/метрики (отдельный скилл `instagram-analytics`, Apify).
- Не использовать meta-prompt (скилл процедурный).
- Не делать OCR слайдов / транскрипт рилса в этом скилле (опираемся на подпись; OCR/ASR — позже, опц. через image-скилл).
