# Фикс границ reels/carousel + формат-история — дизайн

**Дата:** 2026-06-30
**Статус:** утверждён (брейншторм)
**Тип:** правка инструкций скиллов (не код) + правка приватного `content-studio/CLAUDE.md`

## Проблема (из практики, по фактам)

Переупаковка рилса Ника Сараева («Find Skills») в content-studio дала: стену текста, тело целиком захвачено прогрев-фреймом автора («ты прослойка / мина»), навязчивый «твист»-разоблачение даже на чистом showcase, и много итераций до результата уровня Ника. Диагноз (со ссылками на исходник):

1. **Прогрев течёт в тело.** `reels-script` [SKILL.md:5] «удар по болям зрителя» (двусмысленно) + [:65] «прогревочный — продажа через боль жёстче» + яркая боль Хормози в `content-studio/CLAUDE.md`. Нет границы «тело не продаёт» → продуктовая боль автора заливает всё тело.
2. **Автотвист.** Список углов [:29] содержит «разоблачение/провокация» + бренд «вайб-кодинг опасен» → модель контравертит даже showcase. Showcase/giveaway не оформлен как самостоятельный угол без твиста.
3. **Длина/текстура.** Цель [:67],[:90] «200–300 слов / 60 сек» → стена; при «урежь» — рубит механически, выходит «обрезано». Нет эталона панчлайн-подачи.

База хуков/подходов НЕ лечит сбои 1–2 — это баг границ/дефолтов. Поэтому первым — фикс границ. База подходов (`тип: script`) отложена (ловим итеративно).

## Решения (утверждено)

### A. reels-script SKILL.md

1. **Принцип «тело не продаёт».** Тело останавливает скролл и даёт ценность по ТЕМЕ; не несёт позицию/продукт/прогрев автора. Оффер/позиционирование — только в CTA, коротко, по уместности. Модель Ника: продажа вниз по воронке, не в ролике.
2. **Снять двусмысленность «боль зрителя».** Боль = боль зрителя про *тему ролика*, не продуктовая боль автора (прослойка/мина).
3. **CTA-интенсивность строго в CTA.** «Продажа через боль жёстче» ограничить CTA; тело не несёт её ни на какой стадии.
4. **Showcase/giveaway — полноценный угол.** Добавить в список углов «обзор-инструмента / giveaway»; зафиксировать: твист не нужен. Твист (разоблачение/провокация) — осознанно выбранный угол, не автодобавка.
5. **Длина — гибкая через эталон.** Убрать жёсткие 200–300 слов/60 сек как дефолт. Принцип: длина следует эталону виральной короткой формы (как правило 20–40 сек, до 60 только для контент-тяжёлых углов); решает удержание, не счётчик слов. Каждая строка тела = панчлайн (без воды, без слов, лишь описывающих конкретику вместо показа).
6. **6-й тест-страж:** «тело не продаёт и не содержит позицию автора (только CTA)». Провален — переписать тело.
7. **Хук — не выдумывать.** Явно: приоритет hook-base (рабочие паттерны), сочинять с нуля — крайний случай (уже в порядке резолва, усилить формулировкой).

### B. content-studio/CLAUDE.md (вне git, приватный)

- Боль Хормози ограничить строго CTA (остаётся спецификацией призыва, не топливом тела).
- Добавить: тело рилса/карусели = ценность по теме, ноль продажи/позиции; оффер/прогрев — только CTA, коротко, по уместности; Ник = продажа вниз по воронке.

### C. carousel-script SKILL.md

1. **Тот же фикс границ:** строка 73 «продажа через боль жёстче» → строго CTA-слайд; слайды-ценность не продают. Принцип «тело не продаёт» + развязка «боль зрителя про тему ≠ продуктовая боль автора».
2. **Формат-история (объяснялка).** Добавить угол «обучающая история / объяснялка» рядом с перечислением (пример: «что такое скиллы / MCP, ты делаешь это-это» — как пример, НЕ жёсткий шаблон).
3. **Усилить тягу.** На повествовательном/историческом формате каждый слайд заканчивается крючком на следующий — open loop / интрига / мини-клиффхэнгер (как в сериалах), не «листай дальше»; «нить» (разрез фразы, PR#8) — один из приёмов, не единственный; стрелка опциональна. Расширяет существующую [SKILL.md:68], не заменяет.
4. **Аватар не зашивать.** Новички в AI-разработке — мягкий дефолт-ориентир тона, легко перебить (автор ещё сужает аудиторию).
