# Дизайн скилла `ai-research` (ресёрч AI-мира, мини-штаб)

Дата: 2026-06-18
Статус: утверждён, готов к плану реализации

## 1. Цель

Скилл собирает и курирует новости/тренды AI-мира из авторитетных источников и выдаёт результат в одном из трёх режимов. Это **мини-штаб** внутри большого агента-ассистента: `SKILL.md` роутит между режимами по общим `references/`.

Результат — текст (идеи / дайджест / недельная подборка) как артефакт. Скилл tool-agnostic: ресёрчит тем, что доступно (Perplexity MCP предпочтительно), и спроектирован так, чтобы подключение тулов не требовало переписывания.

## 2. Границы (scope)

**Входит:** сам скилл `ai-research` (ресёрч + курирование, 3 режима вывода).
**НЕ входит:** постинг/рассылка; конфиг большого агента (Hermes-персона/роутинг — отдельный тонкий слой на рантайме); реальные MCP-серверы (подключаются в рантайме, скилл лишь предпочитает их).

## 3. Личность и мини-штаб

`ai-research`. `SKILL.md` = роутер по режимам. Режимы и форматы выхода — в `references/output-modes.md`; список источников — в `references/sources.md`. При «обе»/составных запросах режимы могут комбинироваться.

## 4. Режимы вывода (роутер)

Определять из запроса; уточнять у пользователя только если режим неясен.

| Режим | Когда | Выход | Куда дальше |
|---|---|---|---|
| `ideas` (дефолт) | «найди идеи про…», «о чём снять» | ранжированные кандидаты | → reels-script / carousel-script |
| `digest` | «собери дайджест», «лучшие новости для клуба» | курированный дайджест | → пост / рассылка клубу |
| `weekly` | «подборка недели», «главная новость каждого дня» | 7 дней (Пн–Вс), главная новость дня | → **carousel-script** |

## 5. Source-лесенка (tool-aware, гибрид)

Порядок разрешения исследовательского источника:
1. **Perplexity MCP** — если подключён (лучший живой ресёрч).
2. **WebSearch / WebFetch** — встроенный фоллбэк.
3. **Evergreen-генерация** — если живой поиск недоступен/не нужен: вывести вечнозелёные боли/углы из темы.

Гибрид: живой поиск свежих углов/новостей + добор вечнозелёных паттернов. Ключ Perplexity — в `.env.template` (`PERPLEXITY_API_KEY`).

## 6. Источники (`references/sources.md`)

Курированный список авторитетных AI-источников по тирам (редактируемый):
- **Tier-1 медиа:** TechCrunch (AI), The Verge (AI), Ars Technica, VentureBeat, MIT Technology Review, Wired.
- **Блоги лабораторий/компаний:** OpenAI, Anthropic, Google DeepMind, Google AI, Meta AI, Microsoft, AWS ML Blog, Hugging Face, Mistral, NVIDIA.

## 7. Форматы выхода режимов (`references/output-modes.md`)

- **`ideas`** — ранжированный список кандидатов, каждый: `тезис · угол · почему зайдёт (боль/тренд) · формат (рилс/карусель/обе) · источник (ссылка)`.
- **`digest`** — топ-N новостей: заголовок · 1–2 строки сути · ссылка · «почему важно». Курировано под клуб.
- **`weekly`** — 7 строк (Пн–Вс), каждая: день · главная новость дня · 1 строка · ссылка. Это готовый хэндофф-структура для `carousel-script` (каждый день = слайд).

## 8. Антигаллюцинации (критично для новостного тула)

- **Всегда ссылка на источник** для каждой новости/идеи, опирающейся на факт.
- **Не выдумывать** новости, цифры, цитаты, даты. Нет подтверждения — не включать или явно пометить «нужно проверить».
- Для `digest`/`weekly` — только реальные, найденные в источниках новости (никакой генерации фактов).
- Опираться на **текущую дату** (актуальность); за свежими новостями идти в живой поиск, не по памяти.

## 9. Ограничения и переиспользование

- Ниша — AI по умолчанию; тема — параметр; поддержка кросс-нишевого переноса.
- Дисциплина ответа — как у всех скиллов хаба: рассуждения молча, без финального CTA к диалогу.
- DB-aware шов: идеи/паттерны позже могут питать общую базу хуков.
- Ядро инструкции — через `meta-prompt`; упаковка через `skill-creator`. После — глобальная установка (`skills/install-skills.*`) + витрина `skills/README.md`.

## 10. Файлы

```
skills/ai-research/
├── SKILL.md            # роль, режимы-роутер, source-лесенка, антигаллюцинации, дисциплина, ссылки

└── references/

```

## 11. Non-goals (YAGNI)

- Не постим и не рассылаем (это другой слой/тул).
- Не дробим режимы на отдельные скиллы — один скилл, режимы внутри (мини-штаб).
- Не настраиваем сам MCP-сервер и не пишем код интеграции — скилл лишь предпочитает Perplexity MCP, если он есть в рантайме.
- Не конфигурируем большого агента (Hermes) — это тонкий слой на рантайме, вне этого скилла.
