# Brand Voice Skill — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Собрать скилл `skills/brand-voice/` — переиспользуемый артефакт «как звучит мой голос» + универсальные принципы построения текста, подключаемый глобально и читаемый по пути другими агентами.

**Architecture:** Скилл-папка из тонкого `SKILL.md` (рабочий контракт) и двух обычных `.md` в `references/` (голос отдельно, принципы отдельно). Ядро голоса генерируется через `meta-prompt` из реальных постов автора; принципы дистиллируются из руководства автора. Упаковка и eval-прогон — через `skill-creator`.

**Tech Stack:** Markdown; Claude Code skills (`SKILL.md` + frontmatter `name`/`description`); `meta-prompt`, `skill-creator` (скиллы хаба); `install-skills.ps1` (junction-подключение на Windows).

## Global Constraints

- Язык всех артефактов — русский (как у остальных скиллов хаба).
- Голос и принципы — **раздельные** файлы в `references/`, обычный текст, читаемый по пути.
- Каждый новый скилл добавляется в каталог `skills/README.md` и подключается через `install-skills.ps1` (CLAUDE.md хаба).
- Создание контентного ядра — через `meta-prompt`, вывод — сырьё, адаптируется (CLAUDE.md хаба).
- **Принцип №1 (анти-шаблонизация):** словарь = регистр, не чек-лист; юмор сохраняем, не форсируем; эмодзи дозированно.
- **Принцип №2 (минимальное вмешательство):** лестница входов (сырьё / полуфабрикат / почти готовое); Ильяхов = фильтр, не переписчик; при конфликте побеждает автор.
- **Принцип №3 (критика по светофору):** 🟢 сохранить · 🟡 склоняться к автору · 🔴 честно «так не будет».
- **Telegram-вёрстка:** эталон 2-строчных абзацев + пустая строка; гибко, с исключениями (списки, шаги, цитаты), в рамках здравого смысла. Не догма.
- Работа в ветке `brand-voice`. Спек: `docs/superpowers/specs/2026-06-23-brand-voice-design.md`.
- Скилл НЕ постит/не доставляет контент (слой рантайма).

## Task 2: `writing-principles.md` — универсальные принципы

Независим от извлечения голоса: основан на «Руководстве по написанию постов» автора (есть на руках). Делаем первым — он стабилен и задаёт предохранитель редактуры, на который опирается SKILL.md.

**Files:**
- Create: `skills/brand-voice/references/writing-principles.md`

**Interfaces:**
- Produces: документ-чек-лист с секциями 1–6 (см. спек). Секция 6 «Граница редактуры» — на неё ссылается SKILL.md (Task 4).

- [ ] **Step 1: Зафиксировать критерии приёмки (acceptance)**

Документ принят, если: (1) покрывает 6 секций спека; (2) содержит §6-предохранитель дословно про «фильтр, не переписчик» + «побеждает автор» + светофор; (3) нет голых оценок без примеров (сам соответствует своим правилам); (4) автор подтверждает.

- [ ] **Step 2: Написать `writing-principles.md`**

Дистилляция руководства автора в рабочий чек-лист для агента. Секции (из спека):
1. Базовый принцип — текст для читателя; полезное действие.
2. Чистка — стоп-слова; оценки → факты; глаголы вместо отглагольных; действительный залог.
3. Синтаксис — одна мысль на предложение; не двигать «камеру» назад; осторожно с однородными.
4. Структура — заход цепляет; подзаголовки со смыслом; концовка не обрывается.
5. Приёмы силы — абстракция + пример; от обратного; уровень подготовки читателя.
6. **🛑 Граница редактуры** — применять как фильтр читаемости минимальным вмешательством; не переписывать смысл/идеи/акценты/характерные фразы; конфликт «чище по Ильяхову» vs «так сказал автор» → побеждает автор; критика по светофору (🟢/🟡/🔴), в 🔴 — прямо «так работать не будет».

Каждое правило — с коротким примером «было → стало» из руководства (не абстракции).

- [ ] **Step 3: Самопроверка против acceptance**

Перечитай: все 6 секций на месте? §6 содержит три элемента (фильтр-не-переписчик, побеждает-автор, светофор)? Нет ли в самом тексте канцелярита/голых оценок? Исправь инлайн.

- [ ] **Step 4: Гейт вычитки автором**

Покажи автору `writing-principles.md`. Спроси: «§6-предохранитель сформулирован так, чтобы агент НЕ переписывал твой замысел, но честно критиковал в красной зоне?» Внеси правки.

- [ ] **Step 5: Commit**

```bash
git add skills/brand-voice/references/writing-principles.md
git commit -m "brand-voice: writing-principles.md — принципы + предохранитель редактуры"
```

## Task 3: `brand-voice.md` — голос автора (через meta-prompt)

**Files:**
- Create: `skills/brand-voice/references/brand-voice.md`
- Read: `docs/superpowers/working/brand-voice-corpus.md` (из Task 1)

**Interfaces:**
- Consumes: чистый corpus постов (Task 1).
- Produces: документ голоса, секции 1–12 (см. спек). На него ссылается SKILL.md (Task 4) и будущие агенты.

- [ ] **Step 1: Зафиксировать критерии приёмки**

Документ принят, если: (1) 12 секций спека; (2) §4 Словарь подан как РЕГИСТР с анти-правилом (не чек-лист), §6 Юмор «сохраняй-не-форсируй», §7 эмодзи-дозировка, §8 Telegram-вёрстка с исключениями, §10 «Библиотека образцов (растёт)» — НЕ фиксированный эталон, §12 «сохранять авторские формулировки»; (3) автор читает и говорит «звучит как я, и нет навязанных слов».

- [ ] **Step 2: Анализ постов по 5 осям**

Прочитай corpus. Выпиши по осям (метод блогера), опираясь на реальные цитаты:
- **Тон** — разговорно-экспертный, уверенный, лёгкая ирония без жести.
- **Структура предложений** — mix, доминанта коротких; удар короткой после длинной.
- **Словарь-регистр** — какие слова/обороты реально встречаются (примеры: «зуб даю», «комон», «по факту», «спойлер», «движуха»); пометить как регистр, не список.
- **Приёмы** — миф → разбор; личный опыт; риторические вопросы; числа-якоря; метафоры; вовлекающие концовки.
- **Что НЕ используется** — канцелярит, инфоцыганщина, заискивание.
Сохрани это как черновое ядро (scratch, не файл скилла).

- [ ] **Step 3: Прогон чернового ядра через `meta-prompt`**

Вызови скилл `meta-prompt`, подав ему: черновое ядро по осям + три сквозных принципа + структуру 12 секций из спека. Задача meta-prompt — превратить это в грамотную инструкцию голоса с **формулировками, гасящими шаблонизацию** (Принцип №1): словарь как регистр, анти-правило «не вставляй намеренно / не >N маркеров / нет в черновике — не добавляй / не повторяй из поста в пост».

- [ ] **Step 4: Адаптировать вывод meta-prompt в `brand-voice.md`**

Вывод meta-prompt — сырьё (CLAUDE.md). Адаптируй под 12 секций спека, добавь шапку «живой документ, v0, докручивается». Раздел §10 «Библиотека образцов» — наполни 3–5 короткими выдержками из corpus, явно подписав «ориентир по звучанию, не эталон; пополняется».

- [ ] **Step 5: Самопроверка против acceptance + анти-шаблон**

Перечитай: §4 точно не выглядит как список-вставка? §6 юмор не предписан везде? §8 вёрстка с исключениями? §10 без слова «эталон» в смысле «образец для копирования»? Исправь инлайн.

- [ ] **Step 6: Гейт вычитки автором (ключевой)**

Покажи автору `brand-voice.md`. Прямые вопросы: «Звучит как ты? Нет ли слов, которые модель будет навязчиво пихать? §8-вёрстка не превратилась в догму?» Внеси правки, повтори при необходимости.

- [ ] **Step 7: Commit**

```bash
git add skills/brand-voice/references/brand-voice.md
git commit -m "brand-voice: brand-voice.md — голос автора (через meta-prompt), v0"
```

## Task 6: Поведенческая проверка (eval-прогон)

«Тест» скилла: прогнать на реальных сценариях и убедиться, что три принципа соблюдаются. Через подход `skill-creator` (eval-кейсы).

**Filesः**
- Working (не коммитим): `docs/superpowers/working/brand-voice-evals.md` — кейсы и результаты.

- [ ] **Step 1: Описать eval-кейсы**

Запиши кейсы с ожидаемым поведением:
1. **Триггер** — «напиши пост в телегу про X» → скилл активируется, читает оба references.
2. **Вход «сырьё»** — «опиши вот эту новость: …» → строит текст сам, голос + вёрстка.
3. **Вход «почти готовое»** — дать почти готовый абзац → ЛЁГКОЕ касание, формулировки автора сохранены (НЕ переписан).
4. **Анти-шаблон** — 3 поста подряд на разные темы → разговорные маркеры НЕ повторяются, не пихаются в каждый; юмор не форсирован.
5. **Вёрстка** — длинный текст → 2-строчные абзацы; но список/цитата НЕ раздроблены.
6. **Светофор 🔴** — дать заведомо плохой каркас (абсурд) → агент честно критикует «так не будет», а не поддакивает.

- [ ] **Step 2: Прогнать кейсы**

Выполни каждый кейс в свежем контексте (или попроси автора прогнать). Зафиксируй фактическое поведение в `brand-voice-evals.md`.

- [ ] **Step 3: Сверить с ожиданиями, исправить артефакты**

Где поведение разошлось — правь соответствующий `references/*.md` или `SKILL.md` (чаще всего: усилить анти-правило в §4, уточнить лестницу/светофор). Перепрогнать упавшие кейсы.

- [ ] **Step 4: Гейт автора**

Покажи автору результаты прогона (особенно кейсы 3, 4, 6 — его боли). Подтверждение или правки.
