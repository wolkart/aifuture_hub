# Интеграция content-machine → post-hq + ai-research — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Подключить `post-hq` к оркестратору `content-machine` как третий формат доставки «пост» и дотянуть хэндофф `ai-research → post-hq`, чтобы найденная идея/новость уходила в написание текстового поста без ручного переноса.

**Architecture:** Контентные скиллы — это Markdown-инструкции (`SKILL.md` + `references/`), не код. «Реализация» = точечные правки текста скиллов по утверждённому контракту; «тесты» = поведенческий eval-прогон в свежем контексте + гейт автора. `content-machine` остаётся тонким оркестратором (DRY): он только добавляет «пост» в рекомендацию формата и делегирует `post-hq`, который сам разруливает площадку × цель × голос × review. Содержимое модулей дословно не дублируется между скиллами.

**Tech Stack:** Markdown; Claude Code Skills (`SKILL.md`, `references/`); git; ручной поведенческий eval (без автотестов — у прозаических скиллов их нет, как и в прошлых циклах хаба).

## Global Constraints

- **DRY:** `content-machine` НЕ повторяет логику суб-скиллов; площадку/цель/голос/фреймворки/review знает только `post-hq` — единый источник правды.
- **Граница «пост»:** в `content-machine` формат «пост» = самостоятельный текстовый артефакт (Telegram / Threads / X / LinkedIn). Instagram-подпись остаётся внутри `post-hq` (`instagram-caption`) как компаньон визуала и из `content-machine` как формат НЕ дёргается.
- **`content-machine` не парсит площадку:** если драйвер задал площадку — пробрасывается в `post-hq` строкой как есть (пасс-тру); машина её не выводит.
- **`post-hq` функционально не меняется:** он уже принимает сырьё и сам роутит. Правки post-hq — только документационная строка (Task 3).
- **Ветка:** работаем в `content-machine-post-hq` (уже создана; спек закоммичен). На `main` напрямую не пишем.
- **Коммиты:** в конце каждого таска; сообщение оканчивается `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.
- **Без автотестов:** шаги «verify» для правок Markdown — это `grep`/чтение на присутствие нужного текста; настоящая приёмка — Task 4 (eval + гейт автора).
