# Reels-Script Skill — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Собрать скилл `skills/reels-script/`, который превращает идею в готовый вирусный сценарий Reels/Shorts по утверждённому спеку.

**Architecture:** Скилл = `SKILL.md` + `references/` (хребет виральности, форматы, схема паттернов с сид-хуками). Ядро инструкции генерируем через скилл `meta-prompt`, упаковываем и проверяем через `skill-creator`. «Тесты» — eval-петля: прогон тест-промптов со скиллом, объективные проверки (наличие битов, гейта ширины, лидмагнита, метастроки) + качественный обзор.

**Tech Stack:** Markdown, скиллы `meta-prompt` и `skill-creator`, субагенты для прогонов.

**Verification model:** Поскольку артефакт — текст инструкции, «провал теста» = прогон тест-промпта, где вывод не содержит обязательных элементов из чек-листа (§ Task 7). Проверки выполняются субагентом-грейдером или скриптом по выводам.

**Спек:** [docs/superpowers/specs/2026-06-17-reels-script-skill-design.md](../specs/2026-06-17-reels-script-skill-design.md)

- [ ] **Step 1: Написать файл** со всеми блоками из спека §6:
  - **Формулы хуков** — провокация / неожиданность / интрига / отрицание / обещание, с 1 примером каждая; требования: ≤10 слов, понятно без контекста, бьёт в эмоцию.
  - **Атомизация (Хормози)** — раскладывать совет до мельчайших наблюдаемых действий-«атомов»; атом ↔ кадр.
  - **Абстрактное → наблюдаемое** — запрет на невидимые качества («будь экспертным»); только видимые действия («добавь фразу X»).
  - **Инверсия (опц.)** — перечисли, что гарантированно проваливает, и переверни — генератор хуков для угла «разоблачение».
  - **Удержание** — open loop, темп, ноль воды.
  - **Расширение темы / уход от жаргона.**
  - **CTA / лидмагнит-паттерны** — мягкий CTA vs полноценный лидмагнит с кодовым словом.

- [ ] **Step 2: Проверка** — прочитать файл, убедиться, что каждый блок §6 присутствует и содержит конкретные примеры (не абстракции). Намеренно НЕ включены мотивационные темы Хормози.

## Task 7: Создать eval-кейсы

**Files:**
- Create: `skills/reels-script/evals/evals.json`

- [ ] **Step 1: Записать 4 тест-промпта** (`evals/evals.json`) — реалистичные:
  1. **Расплывчатая идея:** «сделай рилс про то что нейросети помогают экономить время»
  2. **Идея + готовый хук:** «рилс, тезис: люди боятся что ИИ заберёт работу. Хук уже есть: "ИИ не заберёт твою работу — её заберёт тот, кто умеет с ним работать"»
  3. **Переупаковка кросс-ниша:** «вот вирусный фитнес-рилс с хуком "ты тренируешься годами и не растёшь — вот почему". переупакуй под нишу ИИ»
  4. **Разговорный формат:** «разговорный рилс, показываю с телефона как я за 2 минуты делаю пост через Claude»

- [ ] **Step 2: Зафиксировать чек-лист проверок (assertions)** для грейдинга:
  - Содержит все 5 битов (Хук, Усиление, Контент, Вывод, Лидмагнит).
  - Присутствует тест/гейт ширины (5М+).
  - Хук ≤10 слов.
  - Есть лидмагнит или мягкий CTA.
  - Есть метастрока в схеме паттернов (ниша/угол/манера/тип хука).
  - Нет клише из стоп-листа («в современном мире», «не секрет»…).
  - Кейс 2: хук пользователя сохранён ИЛИ отвергнут с конкретной причиной (не подменён молча).
  - Кейс 3: результат — не копия, сменены примеры/угол; помечена кросс-нишевость.
