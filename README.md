# AI Future Hub

Хаб полезных материалов, которыми удобно делиться. Каждая папка — отдельный раздел: скиллы, автоматизации, шаблоны и другие классные штуки.

Любой материал можно **скачать архивом** (кнопка `Code → Download ZIP`) или **склонировать** репозиторий целиком:

```bash
git clone https://github.com/wolkart/aifuture_hub.git
```

## Под капотом: как устроен харнес

Это не куча разрозненных промптов. Это **харнес** — система слоёв, где каждый скилл делает одно дело, а рантайм собирает их под конкретную задачу. Принципы, по которым он построен:

**1. Слои, а не монолит.** Голос автора, принципы текста, цель (контент / продажа), площадка (Telegram / Threads / LinkedIn) — это разные слои. Каждый живёт отдельно и правится независимо. Меняешь голос — не трогаешь принципы; добавляешь площадку — не переписываешь остальное.

**2. Композиция, а не дублирование.** Скиллы ссылаются друг на друга, а не копируют. Голос описан один раз в скилле `brand-voice` — и любой пишущий скилл берёт его как единый источник правды. Нужны карточки к посту — штаб делегирует `carousel-script`, а не повторяет его логику.

**3. Не дампить — дистиллировать.** Знание (Хормози, Огилви, «Пиши, сокращай») не вываливается в модель целиком. Из каждой книги извлекается компактный чек-лист приёмов «к применению». Модель и так знает канон — мы добавляем конкретику и приоритеты, а не 60 страниц пересказа.

**4. Условная загрузка (progressive disclosure).** Подтягивается только то, что нужно задаче. Продающий пост загружает фреймворки Хормози; обычный — нет. Контекст остаётся чистым → модель точнее, быстрее и дешевле.

**5. Голос — последним фильтром.** Любой фреймворк даёт скелет убеждения. Сверху всегда накладывается твой голос: переозвучивает текст в твой стиль и срезает шаблонность. Скелет чужой — кожа твоя.

**6. Штаб из ролей.** Сложный скилл — это не один райтер, а штаб: оркестратор распределяет задачу, роли-агенты пишут под площадку и цель, а роль-ревьюер честно проверяет результат и возвращает на доработку.

**Почему это работает.** Узкий релевантный контекст держит модель острой — раздутый роняет и качество, и цену. Слои эволюционируют независимо, поэтому систему легко расширять, ничего не ломая. А «голос последним фильтром» гарантирует: что бы ни подмешалось внутри, на выходе звучишь ты, а не нейросеть.

**Вживую:** `brand-voice` (готов) — голос как переиспользуемый слой с защитой от шаблонности и критикой по «светофору». `post-hq` (штаб постов, в сборке) — оркестратор, который ведёт идею через голос → цель → площадку → ревью ровно по этим принципам.

## Под себя: подключить свой контент

Скиллы здесь **обезличены и переиспользуемы** — в них нет ничьей ниши, голоса или ключей. Личное живёт у тебя, не в репозитории:

- **Кто ты / голос / позиционирование** — в твоём `~/.claude/CLAUDE.md` (виден в любом проекте) и в скилле `brand-voice` (единый источник голоса).
- **Рантайм контента** — отдельная папка-студия со своим `CLAUDE.md`: твоя ниша, путь к твоему Obsidian-волту, лид-магнит, конвенция «куда писать и как линковать». Скиллы вызываются оттуда, артефакты уходят в твой волт.
- **Ключи доступа** (Apify, OpenAI и т.п.) — в `.env` рядом с конкретным скиллом (всегда gitignored).

Чтобы адаптировать харнес: склонируй репо, подключи скиллы (ниже), заведи свою студию и волт, пропиши в них себя — менять сами скиллы не нужно.

## Как пользоваться

1. Зайди в нужную папку — внутри будет свой `README.md` с описанием.
2. Скачай весь репозиторий архивом или склонируй, либо забери только нужную папку.
3. Следуй инструкциям внутри конкретного материала.

## Как подключить скиллы к Claude Code

Папка `skills/` — это **дистрибутив**: чтобы скилл срабатывал в Claude Code, его нужно положить туда, где Claude ищет скиллы. Удобнее всего — **ссылкой** на склонированный репозиторий (правишь/обновляешь репо — скилл сразу актуален), не копируя файлы.

### Все скиллы сразу (рекомендую)

Скрипты идемпотентны — подключают новые скиллы и не трогают уже подключённые. Запускай повторно после `git pull`, когда в репо появятся новые скиллы.

```powershell
# Windows (без прав админа)
git clone https://github.com/wolkart/aifuture_hub.git
powershell -ExecutionPolicy Bypass -File .\aifuture_hub\skills\install-skills.ps1
```

```bash
# macOS / Linux / Git Bash
git clone https://github.com/wolkart/aifuture_hub.git
bash ./aifuture_hub/skills/install-skills.sh
```

### Один скилл вручную

```powershell
# Windows — junction (mklink /J): работает без админа и Developer Mode
cmd /c mklink /J "$env:USERPROFILE\.claude\skills\reels-script" "C:\путь\к\aifuture_hub\skills\reels-script"
```

```bash
# Linux / macOS / Git Bash
ln -s "$(pwd)/aifuture_hub/skills/reels-script" ~/.claude/skills/reels-script
```

После этого скилл доступен **во всех проектах**. Для одного проекта вместо `~/.claude/skills/` используй `.claude/skills/` внутри него.

> **Перенёс репозиторий?** Ссылки хранят абсолютный путь и станут битыми. Удали их из `~/.claude/skills/` (`Remove-Item <ссылка>` / `rm <ссылка>` — файлы скилла в репо это не трогает) и запусти скрипт установки заново.

> Установку «в один клик» (через `/plugin`) добавлю позже, когда оформлю хаб как плагин-маркетплейс.
